# Архітектура проєкту Garage24

Цей розділ описує технічний устрій системи, патерни проєктування та стандарти розробки.

## 🚀 Технологічний стек
- **Backend:** PHP 8.2+, Laravel 10+
- **Frontend:** React 18 (SPA), Tailwind CSS, Vite
- **Database:** MySQL 8.0 (Relational)
- **Cache/Queue:** Redis
- **Integrations:** Uklon Partner API, Telegram Bot API

---

## 🏗 Ключові концепції

### 1. Мультитенінтність (Multitenancy)
Система спроєктована для обслуговування багатьох незалежних автопарків в межах однієї бази даних.
- **Ізоляція:** Кожна таблиця, що містить дані парку (авто, водії, транзакції), має колонку `park_id`.
- **Механізм:** Використовується `App\Models\Scopes\ParkScope`, який автоматично додає `WHERE park_id = ?` до всіх запитів.
- **Контекст:** Поточний парк визначається сервісом `App\Services\CurrentPark`.

### 2. Права доступу (RBAC)
Використовується розширена модель ролей на базі `spatie/laravel-permission`:
- **Owner:** Повний доступ до парку + білінг.
- **Admin:** Операційне управління.
- **Dispatcher:** Робота з графіком та замовленнями.
- **Mechanic:** Тільки тех. обслуговування та звернення.
- **Bot:** Системний акаунт для автоматизації.

### 3. Фінансова цілісність
- Всі грошові потоки проходять через єдину модель `Transaction`.
- Використовується **Polymorphic Relations** (`payable_type`, `payable_id`) для зв'язку транзакцій з їхніми джерелами.

## 📂 Структура коду (Backend)
- `app/Models/` — Тільки чисті моделі та зв'язки.
- `app/Services/` — Вся бізнес-логіка (Heavy Services).
- `app/Http/Controllers/Api/` — Тонкі контролери, тільки валідація та виклик сервісів.
- `app/Http/Resources/` — Трансформація даних для API (JSON).

---

## 📂 Структура коду (Frontend)
- `resources/js/pages/` — Екрани системи (відповідають сайдбару).
- `resources/js/components/` — Компоненти повторного використання (UI Kit).
- `resources/js/hooks/` — Кастомні хуки для роботи з API та станом.
