# Garage24 Workspace (Documentation & Meta)

Ця папка є центральним вузлом для документації, робочих матеріалів та інструкцій для розробників та ШІ-асистентів.

## 🏗 Структура проєкту

Проєкт розділений на три основні частини:

1.  **`/app`** — Основний додаток (Система керування автопарком).
    *   **`/app/park-react-spa`** — Моноліт на Laravel 10+ з фронтендом на React.
    *   Ключові технології: PHP (Laravel), JavaScript (React, Tailwind CSS, Vite), MySQL.
2.  **`/landing`** — Лендінг та маркетингові матеріали.
    *   **`/landing/www`** — Написано на Leaf PHP (легковагий фреймворк).
3.  **`/workspace`** — Документація, плани розробки та контекст для ШІ (цей файл).
4.  **`/doc`** — Додаткові скріншоти, PDF та архівні матеріали.

## 🤖 Правила для ШІ-асистентів (LLM Guidelines)

### 1. Мова та комунікація
*   **Основна мова:** Українська. Спілкування та коментарі в документації — українською. Код (змінні, класи) — англійською.
*   **Підтвердження:** ЗАВЖДИ надавай план змін перед їх втіленням. Чекай на явне "так" або "вперед".
*   **Крок за кроком:** Складні завдання розбивай на етапи та підтверджуй кожен окремо.

### 2. Принципи розробки (Backend - Laravel)
*   **Міграції:** Ніколи не змінюй існуючі файли міграцій. Створюй нові.
*   **Пермішени та Ролі:**
    *   Використовується мультитенінтна система прав.
    *   `Owner` (глобальний) має `park_id = NULL` у таблиці `roles`.
    *   При додаванні нових прав (`permission`) обов'язково створюй пару: `module.view` та `deny:module.view`.
    *   Більше деталей у `app/park-react-spa/docs/architecture-multitenancy-permissions.md`.

### 3. Принципи розробки (Frontend - React)
*   **Навігація:** Використовуй `NavigationItems.js` для керування пунктами меню.
*   **Права доступу:** Використовуй функцію `canAccess('module')`, яка автоматично перевіряє і `view`, і `deny` правила.

### 4. Безпека та Процеси
*   **Коміти:** Не роби `git commit` без прямої вказівки.
*   **Документація:** Оновив код — онови відповідний `.md` файл у `/docs` або `/workspace`.
