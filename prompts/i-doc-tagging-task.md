# Інструкція: Розмітка системи i-doc

Ти дієш як Senior Frontend Engineer. Твоє завдання — впровадити систему самодокументування **i-doc** на конкретній сторінці React-додатку.

## 🎯 Мета
Розставити мітки `data-i-doc` та наповнити каталог підказок (hints) для всіх ключових елементів сторінки (сторінка, таби, секції, модальні вікна).

## 📂 Джерела істини (Source of Truth)
1.  **Реєстр ID:** `app/park-react-spa/resources/js/common/i-doc/i-doc.registry.ts`
2.  **Мапінг маршрутів:** `app/park-react-spa/resources/js/common/i-doc/i-doc.mapping.ts`
3.  **Каталог підказок:** `app/park-react-spa/resources/js/common/i-doc/i-doc.hints.ts`
4.  **Документація:** `app/park-react-spa/docs/i-doc.md` та `app/park-react-spa/docs/refactoring-ongoing.md`

## 🚫 Суворі обмеження
1.  **ЗАБОРОНЕНО використовувати інструмент `replace`.** Використовуй тільки `write_file` для повного оновлення файлу.
2.  **Жодних функціональних або візуальних змін.** Не міняй логіку, верстку чи стилі. Тільки додавання атрибутів та пропсів.
3.  **Мова підказок — українська.** Описи мають бути лаконічними, корисними та професійними.

## 🛠 Технічні стандарти
1.  **Конвенція іменування:**
    *   `page:назва-сторінки` — для кореневого елемента сторінки.
    *   `tab:назва-табу` — для контенту всередині табів.
    *   `sec:назва-секції` — для логічних блоків (таблиці, графіки).
    *   `modal:назва-модалки` — для модальних вікон.
2.  **Атрибути:**
    *   Кореневий елемент сторінки повинен мати: `data-i-doc={PageIndices.NAME}` та `data-page-ready="true"`.
    *   Інші елементи: `data-i-doc={...}`.
3.  **Реєстрація:**
    *   Кожен новий ID у `i-doc.registry.ts` повинен мати коментар з відносним шляхом до файлу, де він використовується.
    *   Використовуй `as const` для об'єктів у реєстрі.
4.  **Модальні вікна:**
    *   Переконайся, що кастомна модалка приймає `{...props}` і прокидає їх до базового компонента `Modal` або `TabsModal`.
    *   Видали ручне виведення `<PageHint />` всередині модалок, якщо воно там є — базові компоненти тепер виводять хінт автоматично при наявності `data-i-doc`.

## 🔄 Робочий процес
1.  **Дослідження:** За отриманим URL знайди відповідний файл сторінки (через `app.jsx` маршрути або `grep`). Визнач всі модалки, таби та секції.
2.  **Реєстрація:** Додай нові ID до `i-doc.registry.ts` з відповідними коментарями.
3.  **Мапінг:** Додай новий шлях маршруту до `i-doc.mapping.ts` у об'єкт `PathToIDoc`.
4.  **Підказки:** Напиши описи для кожного нового ID у `i-doc.hints.ts`.
5.  **Імплементація:**
    *   Онови файл сторінки, додавши імпорти реєстру та розставивши мітки.
    *   Онови файли кастомних модалок, якщо вони не прокидають `...props` або мають застаріле ручне виведення хінта.
6.  **Валідація:** Перевір, що всі імпорти вірні і файл синтаксично коректний.
