# Style Guide: Конвенції іменування та плейсхолдери

> Цей файл є довідником з іменування. Опис структури документів та workflow — у `AI_TASK.md`.

---

## 1. Іменування файлів документації

| Тип | Шаблон | Приклад |
| :--- | :--- | :--- |
| Сторінка | `data/[N]-[розділ]/[N]-[назва].md` | `data/02-vehicles/01-list.md` |
| Модальне вікно | `data/shared/modals/[modal_doc_id].md` | `data/shared/modals/fleet_vehicle_form.md` |

**`modal_doc_id`** — формат: `[модуль]_[об'єкт]_[дія]`. Без числових префіксів. Без префіксу `modal_`.

Приклади: `fleet_vehicle_form`, `maint_record_details`, `ops_assignment_form`

## 2. Скріншоти — структура папок та іменування

**Розташування:**

```
screenshots/
  pages/      ← page_*  (повні сторінки)
  tabs/       ← tab_*   (вкладки)
  modals/     ← modal_* (модальні вікна)
  sections/   ← sec_*   (секції та віджети)
```

Темна тема лежить **в тій самій папці** що й світла — суфікс `_dark` їх розрізняє.

**Формула назви файлу:** `[тип]_[модуль]_[об'єкт]_[стан][_dark].png`

Суфікс `_dark` — опціональний, лише для скріншотів темної теми (див. нижче).

### Типи (префікс)

| Префікс | Тип | Пріоритет |
| :--- | :--- | :--- |
| `page_` | Повна сторінка | Найвищий |
| `tab_` | Вкладка сторінки | Високий |
| `modal_` | Модальне вікно | Середній |
| `sec_` | Секція, віджет, окремий елемент | Низький |

### Модулі

Коди модулів (`[module]` у формулі) — визначені в `PROJECT.md`.

### Стани (суфікс)

| Суфікс | Коли використовувати |
| :--- | :--- |
| `main` | Основний вигляд з даними |
| `empty` | Порожній стан (немає даних) |
| `mobile` | Мобільна версія |
| `form` | Чиста форма (модалка) |
| `valid` | Форма з помилками валідації |
| `details` | Режим перегляду (модалка) |
| `filters` | Вікно фільтрів |
| `conflict` | Попередження про конфлікт / дублікат |

### Теми

| Варіант | Суфікс | Обов'язковість |
| :--- | :--- | :--- |
| Світла тема | *(без суфікса)* | **Обов'язково** для всіх скріншотів |
| Темна тема | `_dark` | Опціонально — знімати за наявності темної теми в продукті |

Темний варіант завжди є парою до світлого: реєструється в `_meta/SCREENSHOTS.md` окремим рядком поруч зі своїм основним скріншотом.

### Приклади

```
screenshots/pages/
  page_fleet_list_main.png          ← список авто, світла тема
  page_fleet_list_main_dark.png     ← список авто, темна тема (опціонально)
  page_fleet_list_empty.png         ← список авто, порожній стан
  page_fleet_list_mobile.png        ← список авто, мобільна версія

screenshots/tabs/
  tab_rep_reports_maint.png         ← вкладка ТО на сторінці Звітів
  tab_rep_reports_maint_dark.png    ← та сама вкладка, темна тема (опціонально)

screenshots/modals/
  modal_fleet_veh_form.png          ← модалка додавання авто
  modal_fleet_veh_valid.png         ← модалка авто з помилками валідації
  modal_maint_filters.png           ← модалка фільтрів ТО

screenshots/sections/
  sec_fleet_stats_header.png        ← секція метрик автопарку
```

## 3. Плейсхолдери та якорі

### Плейсхолдери медіа-ресурсів

| Синтаксис | Де ставити | Що означає |
| :--- | :--- | :--- |
| `{{SCRN:id}}` | Будь-який файл документації | Скріншот. `id` — з `_meta/SCREENSHOTS.md`. Тип визначається префіксом `id` |
| `{{MODAL:doc_id}}` | Файли сторінок (в тексті) | Посилання на документацію модалки. `doc_id` — ключ з `MODALS.md`, **без** префіксу `modal_` |

Приклади `{{SCRN:id}}` по типах:

| Плейсхолдер | Що означає |
| :--- | :--- |
| `{{SCRN:page_fleet_list_main}}` | Скріншот повної сторінки |
| `{{SCRN:tab_rep_reports_maint}}` | Скріншот вкладки |
| `{{SCRN:sec_fleet_stats_header}}` | Скріншот секції або віджету |
| `{{SCRN:modal_fleet_veh_form}}` | Скріншот модального вікна |
| `{{MODAL:fleet_vehicle_form}}` | Посилання на документацію модалки |

### Якорі для навігації

Markdown автоматично генерує якорі з заголовків. Кожен `#`-заголовок стає точкою навігації.

**Правило генерації якоря:** заголовок → нижній регістр, пробіли → `-`, спецсимволи видаляються.

| Заголовок у файлі | Згенерований якір |
| :--- | :--- |
| `# Автомобілі (Реєстр флоту)` | `#автомобілі-реєстр-флоту` |
| `## Функціональні можливості` | `#функціональні-можливості` |
| `### Список водіїв` | `#список-водіїв` |
| `### Призначення` | `#призначення` |

**Структурні якорі документації** (де вони знаходяться і що позначають):

| Якір | Утворюється з | Що позначає |
| :--- | :--- | :--- |
| `#огляд` | `## Огляд` | Початок загального опису сторінки |
| `#вкладки` | `## Вкладки` | Розділ з усіма табами |
| `#[назва-вкладки]` | `### Назва Вкладки` | Початок конкретної вкладки |
| `#реєстр-даних` | `## Реєстр даних` | Таблиця даних сторінки |
| `#функціональні-можливості` | `## Функціональні можливості` | Список функцій |
| `#управління-actions` | `## Управління (Actions)` | Розділ кнопок |
| `#технічна-інформація` | `## Технічна інформація` | Блок для розробників |
| `#технічні-особливості` | `## Технічні особливості` | Нюанси реалізації |

**Посилання на якір в іншому файлі:**

```markdown
[Дивись управління](../02-vehicles/01-list.md#управління-actions)
[Вкладка призначення](../03-drivers/01-list.md#призначення)
[Документація модалки](../data/shared/modals/fleet_vehicle_form.md)
```
