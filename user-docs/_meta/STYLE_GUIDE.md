# Style Guide: Технічна документація Park SaaS

## Принципи
1. **Сухий технічний стиль:** Мінімум прикметників, максимум фактів. "Сторінка призначена для..." замість "Чудова сторінка допоможе вам...".
2. **Повнота опису:** Мають бути описані всі поля форм, всі кнопки дій та всі можливі стани (empty state, помилки).
3. **Плейсхолдери:** Використання `{{PAGE:name}}`, `{{MODAL:name}}`, `{{WIDGET:name}}` для посилань на медіа-ресурси.
4. **Структура файлу:**
   - Заголовок (Назва сторінки)
   - Основний плейсхолдер сторінки
   - Призначення (1-2 речення)
   - Функціональні можливості (список)
   - Опис форм/модальних вікон (якщо є)
   - Технічні особливості (пермішени, інтеграції)
