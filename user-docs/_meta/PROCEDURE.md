# Процедура створення документації (Workflow)

Цей документ визначає покроковий алгоритм дій для Агента при описі сторінок та компонентів.

## 1. Дослідження (Research)
- **Код:** Знайти основний React-компонент сторінки, його хуки, сервіси та константи.
- **Архітектура:** Ідентифікувати архітектурний стек (React Query, стани, API методи).
- **Бізнес-логіка:** Перевірити `PAGES_DESCRIPTION.md` та папку `data/` на наявність прихованих правил.

## 2. Візуальна стратегія та Пріоритезація (ОБОВ'ЯЗКОВО)
Перед написанням тексту скласти план скріншотів та **негайно** зареєструвати їх у `_meta/SCREENSHOTS_INDEX.md`.
**Типи та Пріоритети створення:**
1. **Full Page (`page_`):** Основний вигляд, порожній стан (`empty`), мобільна версія (`mobile`).
2. **Modal (`modal_`):** Всі форми (`form`), валідація (`valid`), деталі (`details`), фільтри (`filters`).
3. **Section (`sec_`):** Специфічні блоки (статистика, графіки, віджети).

## 3. Створення контенту (Execution)

### Для Сторінки:
- Створити файл у `data/[розділ]/[ім'я].md`.
- Використовувати сухий технічний стиль.
- Дотримуватися повної структури (див. нижче).

### Для Модального вікна:
- **Перевірка:** Перед створенням перевірити `data/shared/modals/`. Якщо файл є — актуалізувати його.
- **Реєстрація:** Додати/оновити запис у `_meta/MODALS.md` з актуальною датою.
- **Зміст:** Дотримуватися повної структури (див. нижче).

## 4. Метадані та Синхронізація
- Оновити статус у `_meta/INDEX.md` на `✅ Completed`.
- **Дати:** Проставити актуальну дату оновлення (YYYY-MM-DD) в усіх реєстрах (`INDEX.md`, `MODALS.md`, `SCREENSHOTS_INDEX.md`) для кожного зміненого запису.

## СТРУКТУРА ТЕХНІЧНОГО ОПИСУ СТОРІНКИ
1. **Frontmatter** (YAML: route, permission, components).
2. **# Заголовок сторінки**.
3. `{{PAGE:main_screenshot_id}}`.
4. **Призначення** (1-2 речення).
5. **Функціональні можливості** (детальний технічний список).
6. **Реєстр / Таблиця даних** (опис кожної колонки та логіки сортування).
7. **Операційне управління** (опис усіх кнопок дій, глобальних та рядкових).
8. **Технічна інформація (для розробників)** (Таблиця: Файл | Роль — компоненти, хуки, запити, сервіси).
9. **Технічні особливості** (логіка валідації, автоматична синхронізація, нюанси адаптивності).

## СТРУКТУРА ОПИСУ МОДАЛЬНОГО ВІКНА
1. **# Заголовок модалки**.
2. `{{MODAL:screenshot_id}}`.
3. **Призначення**.
4. **Поля форми / Вміст** (Таблиця: Поле | Опис | Технічні деталі: типи вводу, обов'язковість).
5. **Логіка та валідація** (залежності між полями, правила збереження, обробка конфліктів 409).
6. **Технічна інформація** (Таблиця: Файл | Роль — компонент, хуки, API методи).
