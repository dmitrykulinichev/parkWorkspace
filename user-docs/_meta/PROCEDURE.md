# Процедура створення документації (Workflow)

Цей документ визначає покроковий алгоритм дій для Агента при описі сторінок.

## Алгоритм дій
1. **Дослідження (Research):**
   - Знайти основний React-компонент сторінки.
   - Прочитати код компонента, хуків та сервісів.
   - Вивчити існуючий чернетковий опис у папці `data/`.
   - Перевірити `PAGES_DESCRIPTION.md` на наявність прихованої бізнес-логіки.

2. **Аналіз інтерфейсу та Візуальна стратегія:**
   - Виписати всі колонки таблиць, фільтри, кнопки дій.
   - Ідентифікувати всі модальні вікна.
   - **Негайна реєстрація:** Якщо під час аналізу виявлено необхідність у скріншоті, якого немає в `_meta/SCREENSHOTS_INDEX.md`, його потрібно **негайно додати** до реєстру з описом та статусом `📝 Planned`.
   - Зареєструвати всі скріншоти в `_meta/SCREENSHOTS_INDEX.md` перед створенням тексту.

3. **Створення контенту (Execution):**
   - **Сторінка:** Створити/оновити `.md` файл у папці `data/`.
     - Додати Frontmatter (route, permission, main component).
     - Використовувати сухий технічний стиль.
     - Вставити плейсхолдери `{{PAGE:...}}`, `{{WIDGET:...}}`, `{{MODAL:...}}` у відповідних місцях тексту.
   - **Модалки:** Перед створенням опису модального вікна **ОБОВ'ЯЗКОВО**:
     - Перевірити папку `data/shared/modals/` на наявність існуючого файлу.
     - Якщо файл існує — оновити його та дату в реєстрі.
     - Якщо файлу немає — створити новий та зареєструвати в `_meta/MODALS.md`.
     - **Візуалізація модалки:** Запропонувати та зареєструвати скріншот для кожної модалки, якщо це ще не зроблено.

4. **Метадані та Синхронізація:** 
   - Оновити статус у `_meta/INDEX.md` на `✅ Completed`.
   - Проставити актуальну дату оновлення (YYYY-MM-DD) в усіх реєстрах (`INDEX.md`, `MODALS.md`, `SCREENSHOTS_INDEX.md`).

5. **Валідація:**
   - Перевірити відповідність назв скріншотів правилам у `_meta/SCREENSHOTS.md`.
   - Переконатися, що описані всі поля форм та умови їх валідації.
