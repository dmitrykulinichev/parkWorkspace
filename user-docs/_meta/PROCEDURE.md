# Процедура створення документації (Workflow)

Цей документ визначає покроковий алгоритм дій для Агента при описі сторінок та компонентів.

## 1. Дослідження (Research)
- **Код:** Знайти основний React-компонент сторінки, його хуки, сервіси та константи.
- **Архітектура:** Ідентифікувати архітектурний стек (React Query, стани, API методи).
- **Бізнес-логіка:** Перевірити `PAGES_DESCRIPTION.md` та папку `data/` на наявність прихованих правил.

## 2. Візуальна стратегія (ОБОВ'ЯЗКОВО)
Перед написанням тексту скласти план скріншотів та **негайно** зареєструвати їх у `_meta/SCREENSHOTS_INDEX.md`.
**Обов'язкові ракурси:**
- `page_..._list` — основна таблиця з даними.
- `page_..._empty` — вигляд сторінки без даних (placeholder).
- `page_..._mobile` — мобільна версія інтерфейсу.
- `modal_..._form` — чисте вікно форми.
- `modal_..._validation` — форма з відображеними помилками валідації.
- `modal_..._filters` — вікно фільтрації.
- Специфічні стани (напр. `expired`, `active`, `details`).

## 3. Створення контенту (Execution)

### Для Сторінки:
- Створити файл у `data/[розділ]/[ім'я].md`.
- Використовувати сухий технічний стиль.
- **ОБОВ'ЯЗКОВО** додати розділ "Технічна інформація" в кінці.

### Для Модального вікна:
- **Перевірка:** Перед створенням перевірити `data/shared/modals/`. Якщо файл є — актуалізувати його.
- **Реєстрація:** Додати/оновити запис у `_meta/MODALS.md` з актуальною датою.
- **Зміст:** Описати всі поля, логіку залежностей та валідацію.

## 4. Метадані та Синхронізація
- Оновити статус у `_meta/INDEX.md` на `✅ Completed`.
- Проставити актуальну дату оновлення (YYYY-MM-DD) в усіх реєстрах (`INDEX.md`, `MODALS.md`, `SCREENSHOTS_INDEX.md`).

## СТРУКТУРА ТЕХНІЧНОГО ОПИСУ СТОРІНКИ
1. **Frontmatter** (route, permission, components).
2. **# Заголовок сторінки**.
3. `{{PAGE:main_screenshot_id}}`.
4. **Призначення** (1-2 речення).
5. **Функціональні можливості** (список).
6. **Реєстр / Таблиця даних** (опис колонок).
7. **Операційне управління** (кнопки дій та їх наслідки).
8. **Технічна інформація (для розробників)** (Таблиця: Файл | Роль).
9. **Технічні особливості** (валідація, синхронізація, адаптивність).

## СТРУКТУРА ОПИСУ МОДАЛЬНОГО ВІКНА
1. **# Заголовок модалки**.
2. `{{MODAL:screenshot_id}}`.
3. **Призначення**.
4. **Поля форми / Вміст** (Таблиця: Поле | Опис | Технічні деталі).
5. **Логіка та валідація** (залежності полів, правила збереження).
6. **Технічна інформація** (Файл компонента, використовувані хуки та API методи).
