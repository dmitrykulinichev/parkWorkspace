# Команда

{{SCRN:page_park_users_main}}

> Сторінка для управління персоналом парку, призначення ролей та детального налаштування прав доступу.

## Огляд

Розділ "Команда" дозволяє власнику та адміністраторам парку формувати склад користувачів системи, контролювати їхній доступ та допомагати з відновленням паролів. Система підтримує гнучку модель прав, де користувач отримує базові дозволи від ролі, які можна розширити або обмежити індивідуально.

## Реєстр даних

| Колонка | Опис | Логіка / Сортування |
| :--- | :--- | :--- |
| **Ім'я** | Прізвище та ім'я користувача. | Відображає основні дані профілю. |
| **Email** | Електронна пошта користувача. | Використовується як логін. |
| **Роль** | Набір системних ролей користувача. | Позначається кольоровими чіпами (Owner — secondary, Admin — error). |
| **Telegram** | Статус підключення до бота. | Іконка Telegram (синя — підключено, сіра — ні). |
| **Створив** | Користувач, який додав цей запис. | Посилання на картку творця. |
| **Створено** | Дата реєстрації користувача. | Формат DD.MM.YYYY. |

## Функціональні можливості

- **Додавання користувача:** Створення нового аккаунта та відправка запрошення на Email. Використовує {{MODAL:park_user_form}}.
- **Редагування даних:** Зміна імені або ролі існуючого користувача. Використовує {{MODAL:park_user_form}}.
- **Налаштування прав:** Індивідуальне управління дозволами (permissions). Використовує {{MODAL:park_user_permissions}}.
- **Скидання пароля:** Відправка посилання на зміну пароля або повторне запрошення. Використовує {{MODAL:park_user_reset_password}}.
- **Активація через посилання:** Можливість вручну скопіювати лінк активації. Використовує {{MODAL:park_user_activation}}.
- **Фільтрація:** Пошук за роллю, статусом та Telegram. Використовує {{MODAL:park_user_filters}}.

## Технічні особливості

- **Валідація:** При створенні користувача пароль не вводиться — він встановлюється самим користувачем через посилання в листі.
- **Права доступу:** Тільки користувачі з пермішеном `settings.edit` можуть додавати або редагувати членів команди.
- **Пермішени:** Управління детальними правами доступне тільки для користувачів з роллю `Admin` та здійснюється `Owner`-ом.
- **Telegram:** Статус підключення базується на наявності `telegram_id` у моделі користувача.
