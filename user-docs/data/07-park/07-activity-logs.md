# Журнал подій

{{SCRN:page_park_activity_logs_main}}

> Сторінка надає повний аудит дій користувачів та системних подій, необхідний для моніторингу, безпеки та вирішення конфліктів даних.

## Огляд

Сторінка акумулює всі записи про активність у системі. Вона дозволяє фільтрувати події за типами, користувачами або об'єктами, що дозволяє швидко відстежити історію змін даних (хто, коли і які поля редагував).

## Вкладки

### Журнал подій

{{SCRN:tab_park_activity_logs_system}}

> Основний таб, що містить повний перелік системних записів активності.

#### Реєстр даних

| Колонка | Опис | Логіка / Сортування |
| :--- | :--- | :--- |
| **Користувач** | Ім'я ініціатора дії. | - |
| **Дія** | Тип виконаної операції. | - |
| **Опис** | Детальна інформація про подію. | - |
| **IP/Пристрій** | Технічні дані (IP-адреса, браузер). | - |
| **Час** | Дата та час події. | Сортування за часом. Клік → відкриває {{MODAL:activity_log_details}}. |

#### Функціональні можливості

- **Пошук:** Швидкий пошук у журналі за описом або користувачем.
- **Фільтри:** Дозволяють детально налаштувати вибірку подій за типом, датами, користувачами. Використовує {{MODAL:activity_log_filters}}.
- **Статистика:** Аналітика активності за період. Використовує {{MODAL:activity_log_statistics}}.
- **Фільтр критичних подій:** Перемикач, що відображає лише критичні записи, що потребують уваги безпеки.

#### Управління (Actions)

### Глобальні дії (хедер сторінки)

| Дія | Результат |
| :--- | :--- |
| **Фільтри** | Відкриває вікно фільтрації {{MODAL:activity_log_filters}}. |
| **Статистика** | Відкриває модальне вікно {{MODAL:activity_log_statistics}}. |
| **Критичні** | Вмикає режим відображення лише критичних подій. |

#### Технічна інформація

| Файл | Роль |
| :--- | :--- |
| `resources/js/pages/ActivityLogs/index.jsx` | Головний компонент сторінки. |
| `resources/js/pages/ActivityLogs/SystemLogsTab.jsx` | Вкладка "Журнал подій". |
| `resources/js/hooks/useActivityLogsPage.js` | Логіка сторінки та керування станом (React Query). |

## Технічні особливості

- **Валідація:** Фільтрація здійснюється за серверними параметрами.
- **Адаптивність:** Використовує окремі компоненти списку для десктопу та мобільного інтерфейсу.
- **Права доступу:** Доступ до сторінки та критичних подій визначається через дозволи користувача.
