# Журнал розсилок

{{SCRN:page_park_sub_logs_main}}

> Сторінка призначена для моніторингу історії відправки звітів та сповіщень користувачам парку через різні канали (Telegram, E-mail).

## Огляд

Журнал дозволяє оперативно перевіряти статус доставки автоматизованих повідомлень, діагностувати помилки відправки та бачити деталі доставки для кожного користувача.

## Реєстр даних

| Колонка | Опис | Логіка / Сортування |
| :--- | :--- | :--- |
| **Дата/Час** | Дата та час створення запису. | - |
| **Користувач** | Ім'я та email користувача-отримувача. | - |
| **Тип звіту** | Категорія відправленого звіту/сповіщення. | - |
| **Канал** | Канал доставки (Telegram, E-mail). | Візуальна іконка + адреса/ідентифікатор. |
| **Статус** | Результат відправки (Успішно/Помилка). | Відображається чіпом з кольоровою індикацією. |
| **Помилка** | Текст помилки при невдалій відправці. | Тільки при статусі "Помилка". |

## Функціональні можливості

- **Пошук:** Фільтрація за іменем або email отримувача.
- **Тип звіту:** Вибір конкретної категорії розсилки.
- **Канал:** Фільтрація за типом каналу доставки.
- **Статус:** Фільтрація успішних або невдалих розсилок.
- **Оновлення:** Кнопка ручного оновлення списку.
