# Модальне вікно: Деталі авто в Uklon

{{SCRN:modal_intg_uklon_vehicle_details}}

> Перегляд поточного стану автомобіля в системі Uklon. Відкривається кліком на Uklon-іконку в колонці "Інфо" на сторінці `/vehicles`.

## Поля форми

Режим лише для перегляду — поля введення відсутні. Дані отримуються з Uklon API через `uklonService.getVehicle(uklonVehicleId)`.

| Поле | Тип | Обов'язкове | Опис / Технічні деталі |
| :--- | :--- | :---: | :--- |
| **Держ. номер** | text | — | `licencePlate`. |
| **Марка / Модель / Рік** | text | — | `about.maker.name`, `about.model.name`, `about.productionYear`. |
| **Статус** | chip | — | `status.value`: active (зелений) / blocked (червоний). |
| **Рівень комфорту** | chip | — | `uklon.comfortLevel`. |
| **Пріоритет** | chip | — | Показується якщо `uklon.hasPriority = true`. |
| **Брендування** | chip | — | Показується якщо `uklon.hasBranding = true`. |
| **Колір** | text | — | `about.color`. |
| **Тип кузова** | text | — | `about.bodyType`. |
| **Кількість місць** | text | — | `about.seats`. |
| **Опції** | text | — | `about.options` — перелік через кому. |
| **Додано в Uklon** | text | — | `uklon.addedAt`. |
| **Доступно для** | text | — | `uklon.accessibleFor`. |
| **Клас** | text | — | `uklon.comfortLevel` (дублюється в секції "Інформація Uklon"). |
| **Uklon ID** | text | — | `id` — внутрішній ідентифікатор, моноширинний шрифт. |
| **Закріплений водій** | секція | — | `driver.fullName` + `driver.id`. Рендерується тільки якщо `driver` є у відповіді. |
| **Причини блокування** | секція | — | `status.reasons`. Рендерується тільки якщо список не порожній. |
| **Фотоконтроль** | секція | — | Статус, дедлайн, `block_immediately`, причини. Рендерується тільки якщо є. |
| **Період брендування** | секція | — | Статус, дедлайн, Ticket ID. Рендерується тільки якщо є. |
| **Оновлено** | text | — | `synced_at` — час останньої синхронізації з Uklon. |

## Логіка та валідація

- **Режими вікна:** Тільки перегляд. Редагування відсутнє.
- **Обробка помилок:** При помилці завантаження відображається повідомлення з кнопкою "Спробувати знову", яка повторює запит до API.
- **Залежності полів:** Секції "Закріплений водій", "Причини блокування", "Фотоконтроль", "Період брендування" рендеруються умовно — лише якщо відповідні дані присутні у відповіді Uklon API.
