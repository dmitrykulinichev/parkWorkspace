# Модальне вікно: Створити / Редагувати регламент ТО

{{MODAL:modal_maint_schedule_form}}

Форма для визначення правил періодичного обслуговування. На основі цих даних система розраховує прогнозні дати сервісу для всього автопарку.

## Основні поля

| Поле | Опис | Технічні деталі |
| :--- | :--- | :--- |
| **Назва роботи** | Найменування (напр. "Заміна масла"). | Обов'язкове. Використовується для автогенерації системного ID. |
| **Опис** | Детальна інструкція до регламенту. | Опціонально. |
| **Інтервал (км)** | Періодичність обслуговування. | Числове поле. Принаймні один інтервал має бути > 0. |
| **Нагадати за (км)** | Поріг спрацювання попередження. | Наприклад, за 500 км до дедлайну. |
| **Інтервал (роки)** | Часова періодичність. | Кількість років (від 1 до 10). |
| **Нагадати за (днів)** | Поріг часового попередження. | Наприклад, за 14 днів до дедлайну. |
| **Тип двигуна** | Фільтр застосування регламенту. | Всі, Бензин, Дизель, Газ. |
| **Пріоритет** | Важливість роботи. | 1-10 (Критичний), 11-25 (Високий), 26-50 (Нормальний). |
| **Статус** | Перемикач активності (Toggle). | Неактивні регламенти ігноруються системою розрахунку. |

## Логіка та валідація
- **Автоматичний ID:** Поле `maintenance_type` генерується прихованим методом (напр. "Заміна масла" -> `zamina_masla`).
- **Складна валідація:** Система блокує збереження, якщо і поле "км", і поле "роки" залишені порожніми.
- **Підказки:** Під полями інтервалів відображаються динамічні підказки про критичність обраного пріоритету.
