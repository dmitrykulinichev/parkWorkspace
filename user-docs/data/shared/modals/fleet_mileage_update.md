# Модальне вікно: Оновлення пробігу

{{MODAL:modal_fleet_mileage_update}}

Це вікно призначене для регулярної фіксації показників одометра. Воно має вбудовану систему захисту від помилок та конфліктів даних.

## Вкладки (Tabs)

### 1. Історія
Відображає стрічку останніх 20 записів про зміну пробігу:
- Значення пробігу та приріст (різниця з попереднім записом).
- Дата та час надання показань.
- Користувач або водій, який надав дані.
- Коментар (якщо був доданий).

### 2. Оновити пробіг (Форма)

| Поле | Опис | Технічні деталі |
| :--- | :--- | :--- |
| **Новий пробіг (км)** | Актуальний показник одометра. | Має бути більшим за поточний. |
| **Час надання** | Дата та час зняття показників. | Не може бути в майбутньому. За замовчуванням — поточний час. |
| **Передав** | Хто надав інформацію. | Вибір зі списку водіїв. |
| **Коментар** | Додаткова примітка. | Опціонально. |

## Інформаційна панель (Header форми)
- **Поточний пробіг:** Значення в системі на даний момент.
- **Новий пробіг:** Значення, що вводиться.
- **Приріст:** Різниця, що автоматично розраховується (підсвічується зеленим, якщо позитивна).

## Обробка конфліктів та валідація
1.  **Захист від зменшення:** Система видає помилку, якщо новий пробіг менший за останній зафіксований.
2.  **Конфлікт (Warning):** Якщо вводиться значення, яке значно перевищує норму або суперечить іншим записам (Conflict detection), з'являється жовте попередження.
3.  **Примусове оновлення:** У разі конфлікту кнопка "Оновити" змінюється на "Примусово оновити" (червона), що дозволяє менеджеру підтвердити вірність аномальних даних.
