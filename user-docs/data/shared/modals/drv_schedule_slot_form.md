# Модальне вікно: Форма графіку роботи

{{SCRN:modal_drv_schedule_slot_form}}

> Вікно призначене для додавання або редагування слоту (призначення) в графіку роботи. Відкривається при кліку на клітинку графіку або використанні функцій Drag & Drop.

## Поля форми

| Поле | Тип | Обов'язкове | Опис / Технічні деталі |
| :--- | :--- | :---: | :--- |
| **Режим (Тип)** | toggle | Так | Вибір між **Робоча зміна** та **Технічне обслуговування**. |
| **Тип тех. робіт** | radio | Так* | З'являється при виборі тех. обслуговування. Варіанти: **Регламентні роботи** або **Ремонтні роботи**. |
| **Автомобіль** | select | Так | Вибір авто з доступних у парку. Блокується при редагуванні або створенні з контексту конкретного авто. |
| **Водій** | select | Ні | Вибір водія. Можна залишити порожнім для створення вільного слоту. Приховується для тех. обслуговування. |
| **Дата** | date | Так | Дата призначення слоту. |
| **Зміна** | select | Так | Порядковий номер зміни (1–5). При створенні доступна опція **Автоматично (в кінець)**. При редагуванні поле заблоковане. |
| **Примітки** | text | Ні | Додаткова інформація. Плейсхолдер адаптується до вибраного типу робіт. |

## Логіка та валідація

- **Обов'язкові поля:** Автомобіль, Дата, Зміна (або режим Auto), Тип робіт.
- **Режими вікна:**
    - **Створення:** Дозволяє вибирати автомобіль та встановлювати зміну як "Автоматично".
    - **Редагування:** Поля "Автомобіль", "Зміна" та "Режим" заблоковані для зміни цілісності графіку.
- **Технічне обслуговування:** При виборі цього режиму поле "Водій" очищається та приховується, оскільки машина вибуває з лінії.
- **Обробка помилок:** Валідація на бекенді перевіряє конфлікти: один водій не може бути на двох авто одночасно, і на одне авто не можна призначити двох водіїв на ту саму зміну (якщо не використовується функція Force Insert).
- **Минулі періоди:** Якщо увімкнено режим `allowPastEditing`, дозволяється збереження змін за минулі дати.
- **Після збереження:** Викликається рефетч даних графіку через `react-query`.
