# Модальне вікно: Копіювати водія в інший парк

{{SCRN:modal_drv_driver_copy}}

> Вікно копіювання профілю водія до іншого парку в межах мультитенантної системи. Відкривається з пункту "Копіювати в інший парк" контекстного меню на сторінці `/drivers`. Доступно тільки якщо користувач має доступ до декількох парків.

## Поля форми

| Поле | Тип | Обов'язкове | Опис / Технічні деталі |
| :--- | :--- | :---: | :--- |
| **Цільовий парк** | select | Так | Вибір парку призначення. Список — доступні парки з `AuthContext`, крім поточного |
| **Копіювати телефон та email** | checkbox | Ні | За замовчуванням увімкнено. Якщо вимкнено — телефон потрібно вказати вручну після створення |
| **Копіювати Telegram ID** | checkbox | Ні | За замовчуванням вимкнено |
| **Копіювати Uklon ID** | checkbox | Ні | За замовчуванням вимкнено |
| **Копіювати ІПН (РНОКПП)** | checkbox | Ні | За замовчуванням увімкнено |
| **Копіювати екстрені контакти та адреси** | checkbox | Ні | За замовчуванням увімкнено |
| **Копіювати аватар** | checkbox | Ні | За замовчуванням увімкнено |

## Логіка та валідація

- **Обов'язкові поля:** Цільовий парк.
- **Конфлікти:** Якщо водій з таким телефоном/Telegram/Uklon вже існує в цільовому парку — сервер повертає опис конфліктів. Залежно від типу конфлікту форма може пропонувати примусове копіювання (force copy) або вимагати зняти відповідну опцію.
- **Після збереження:** Оновлення списку водіїв поточного парку (refetch). Toast-повідомлення з назвою цільового парку.
