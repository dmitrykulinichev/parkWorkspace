# Модальне вікно: Додати / Редагувати документ

{{MODAL:modal_fleet_document_form}}

Універсальна форма для реєстрації юридичних та технічних документів. Форма автоматично адаптується під обраний тип документа, змінюючи назви полів та набір додаткових параметрів.

## Основні поля форми

| Поле | Опис | Технічні деталі |
| :--- | :--- | :--- |
| **Тип документа** | Категорія (напр. ОСЦПВ, КАСКО, Ліцензія). | Визначає логіку валідації та набір додаткових полів. |
| **Автомобіль** | ТЗ, до якого прив'язується документ. | Заблоковано при редагуванні існуючого документа. |
| **Ким виданий** | Назва органу або страхової компанії. | Лейбл може змінюватися на "Орган що видав". |
| **Номер документа** | Серійний номер бланка. | Лейбл може змінюватися на "Номер Наказу". |
| **Дата видачі** | День підписання/оформлення. | |
| **Дата закінчення** | Термін дії документа. | Можна вимкнути чекбоксом "Безстроково". |
| **Вартість** | Сума витрат на документ. | Автоматично створює фінансову транзакцію. |

## Динамічні параметри (Секція "Додаткова інформація")
Залежно від типу документа, система може вимагати введення специфічних даних (напр. ліміт покриття для страховки або категорія для ліцензії). Ці поля підвантажуються динамічно з конфігурації сервера.

## Логіка та валідація
- **Контроль активних документів:** Якщо ви намагаєтеся додати документ типу, який вже є активним для цього авто (напр. друга страховка ОСЦПВ), система відкриє {{MODAL:confirmation_modal}} з попередженням.
- **Робота з файлами:** До кожного документа можна додати необмежену кількість скан-копій або фото за допомогою вбудованого `FileManager`.
- **Безстроковість:** При встановленні відмітки "Безстроково" дата закінчення видаляється, а документ отримує статус вічно активного.

## Технічна інформація

| Файл | Роль |
| :--- | :--- |
| `resources/js/components/modals/DocumentFormModal.jsx` | Головний компонент форми. |
| `resources/js/queries/documents.js` | Мутації `useCreateDocument` та `useUpdateDocument`. |
| `options.specific_fields` | Масив конфігурації динамічних полів `meta_data`. |
| `force: true` | Параметр API для ігнорування конфлікту активних документів. |
