# Модальне вікно: Додати / Редагувати документ

{{MODAL:modal_docs_form_main}}

Універсальний інструмент для реєстрації юридичних документів автопарку. Форма автоматично підлаштовується під обраний тип документа, пропонуючи відповідні поля та правила валідації.

## Призначення
Створення цифрової копії документа з фіксацією термінів дії, завантаженням скан-копій та можливістю автоматичного обліку витрат на оформлення.

## Поля форми / Вміст

| Поле | Опис | Технічні деталі |
| :--- | :--- | :--- |
| **Тип документа** | Категорія (напр. ОСЦПВ, Техпаспорт). | Обов'язкове. Змінює набір інших полів. |
| **Автомобіль** | ТЗ, до якого належить документ. | Обов'язкове. Вибір зі списку парку. |
| **Ким виданий** | Назва органу або страхової. | Текстове поле. |
| **Номер документа**| Серія/номер бланка. | Текстове поле. |
| **Термін дії** | Дата закінчення. | Можна вимкнути через "Безстроково". |
| **Вартість** | Сума витрат на отримання. | Створює фінансову транзакцію. |
| **Файли** | Скан-копії або фото. | Підтримка декількох файлів (FileManager). |

### Секція "Додаткова інформація" (Динамічна)
{{MODAL:modal_docs_form_dyn}}
Залежно від типу, додаються специфічні поля (напр. "Ліміт покриття" для страхування).

## Логіка та валідація
- **Контроль дублікатів:** При виявленні існуючого активного документа того ж типу відображається попередження {{MODAL:modal_docs_form_force}}.
- **Валідація полів:** При невірному заповненні форма підсвічує помилки {{MODAL:modal_docs_form_valid}}.
- **Безстроковість:** Чекбокс вимикає поле дати закінчення, позначаючи документ як вічно активний.

## Технічна інформація

| Файл | Роль |
| :--- | :--- |
| `resources/js/components/modals/DocumentFormModal.jsx` | Головний компонент форми. |
| `resources/js/queries/documents.js` | Мутації `useCreateDocument` та `useUpdateDocument`. |
| `meta_data` | Об'єкт JSON для зберігання динамічних полів. |
| `options.specific_fields` | Конфігурація полів, що приходить з сервера. |
