# Модальне вікно: Хронологія призначення (Фактична зміна)

{{SCRN:modal_drv_actual_shift_details}}

> Вікно призначене для детального аналізу фактичної активності водія на автомобілі протягом певного періоду. Відкривається при кліку на індикатор фактичної зміни в графіку.

## Опис інтерфейсу

### Шапка
Відображає картку автомобіля та водія, а також точний часовий діапазон призначення (початок — завершення). Якщо зміна ще триває, вказується статус "Триває".

### Хронологія водія
Список подій, безпосередньо пов'язаних з цим призначенням:
- **Призначення:** Початок зміни.
- **Оновлення пробігу:** Фіксація показників одометра водієм.
- **Звіт про проблему:** Створені водієм повідомлення про технічні несправності.
- **Завершення:** Закриття зміни.

Кожна подія містить іконку типу, назву дії, точний час та деталі (наприклад, значення пробігу або опис проблеми).

### Інші події на авто
Окремий блок (якщо є дані), що показує активність на цьому ж автомобілі, яка не належить безпосередньо вибраному водію (наприклад, технічні операції або системні події).

## Логіка та валідація

- **Дані:** Вікно завантажує дані динамічно за ID призначення через API.
- **Синхронізація:** Дані про фактичні зміни зазвичай надходять з інтегрованих систем або через Telegram-бота водія.
- **Стани:** Якщо активності не знайдено, відображається відповідне інформаційне повідомлення.
