# Модальне вікно: Запис про ТО / Ремонт (Заїзд на сервіс)

{{MODAL:modal_maint_record_form}}

Універсальний інструмент для реєстрації сервісних подій. Форма дозволяє групувати декілька різних робіт в рамках одного візиту на СТО для зручності обліку витрат та часу.

## 1. Загальна інформація (Група)

| Поле | Опис | Технічні деталі |
| :--- | :--- | :--- |
| **Автомобіль** | Вибір ТЗ зі списку парку. | При виборі автоматично заповнюється поточний пробіг. |
| **Пробіг (км)** | Показник одометра на момент сервісу. | Валідується на зменшення. Оновлює одометр авто після збереження. |
| **Дата виконання** | Календарна дата візиту. | Використовується для хронології в журналі. |
| **Загальна вартість** | Повна сума, сплачена за сервіс. | Якщо не вказано, система автоматично підсумує вартість усіх робіт нижче. |
| **Файли заїзду** | Кнопка для завантаження документів. | Призначена для загальних чеків, актів виконаних робіт або фото рахунків. |
| **Загальний коментар** | Текстовий опис візиту. | Наприклад: "Плановий візит до офіційного дилера". |

## 2. Секція "Виконані роботи" (Рядки)
Ви можете додати необмежену кількість робіт до одного заїзду за допомогою кнопки **"Додати роботу"**. Кожна робота має свої параметри:

- **Тип роботи:**
    - **ТО (Тех. обслуговування):** Вибір зі встановлених регламентів (напр. "Заміна масла 10к").
    - **Ремонт:** Вільне поле для опису (напр. "Заміна підшипника").
- **Вартість:** Ціна конкретної роботи або запчастини.
- **Пробіг (локальний):** Дозволяє уточнити пробіг для конкретної деталі (якщо відрізняється від загального).
- **Коментар:** Деталі по роботі.
- **Файли роботи:** Кнопка прикріплення документів конкретно до цього пункту (напр. фото дефекту або пакування деталі).

## Логіка та валідація
- **Динамічне управління:** Рядки робіт можна видаляти (якщо їх більше одного).
- **Збереження:** При створенні запису система автоматично генерує фінансову транзакцію та оновлює статус автомобіля (якщо було обрано відповідний тип).
- **Редагування:** При зміні існуючого заїзду система автоматично перераховує загальні суми та оновлює пов'язані файли.
