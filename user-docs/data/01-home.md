# Головна панель (Dashboard)

{{SCRN:page_dash_main}}

> Центральний пульт управління парком: відображає ключові показники, сигнали уваги та стрічку активності.

## Огляд

Головна сторінка агрегує дані з усіх модулів системи і виводить їх у вигляді незалежних віджетів. Кожен віджет відповідає за свій домен: статуси автомобілів, прострочені ТО, проблеми з документами, відкриті задачі та звернення водіїв. Набір відображуваних віджетів налаштовується користувачем — вибір зберігається в `localStorage`. Новий парк бачить блок онбордінгу з картками швидкого налаштування системи.

## Функціональні можливості

- **Онбордінг (Налаштування автопарку):** Відображається, якщо парк ще не налаштований (`/api/dashboard/setup-status`). Показує картки-кроки: додати авто, налаштувати регламенти ТО, додати водія, налаштувати фінанси, Uklon, Telegram, веб-сайт. Кожну картку можна приховати кнопкою `×`. Весь блок можна вимкнути у налаштуваннях дашборду.
- **Пульс парку:** Стовпчастий графік (recharts) активності парку за останні 72 години (додавання авто/водіїв, призначення, поїздки, ТО, ремонти, оновлення пробігу, звернення). Дані — GET `/api/park-pulse?hours=72`.
- **Пульс Uklon API:** Стовпчастий графік активності Uklon API за останні 48 годин. Дані — GET `/api/uklon-pulse?hours=48`. Якщо запит повертає помилку — віджет не рендериться (повертає `null`).
- **Статуси автомобілів:** Перелік автомобілів, згрупованих за статусом (`В роботі`, `Вільні`, `На обслуговуванні`, `В ремонті`, `Неактивні`). Клік на автомобіль → перехід до `/assignments?vehicle_id=...`. Дані — GET `/api/reports/vehicles-dashboard`.
- **Критичні прострочені ТО:** Автомобілі, що перевищили інтервал регламентних робіт, згруповані за типом роботи. Показує надлишок км або днів. Клік на автомобіль → перехід до `/vehicle-maintenance?vehicle_id=...`. Дані — GET `/api/dashboard/critical-overdue-maintenance`.
- **Проблеми з документами:** Авто з відсутньою / простроченою / що закінчується страховкою, техоглядом, ліцензією. Клік на автомобіль → перехід до `/documents?search=[номер]`. Дані — GET `/api/vehicles/document-issues`.
- **Відкриті задачі:** Задачі зі статусом `open` або `in_progress`, згруповані за автомобілем. Показує виконавця, статус, пріоритет. Пряма дія "Завершити" та "Видалити". Використовує {{MODAL:task_form}} для створення нової задачі.
- **Відкриті звіти:** Активні Issue Reports (статус `open`/`in_progress`), згруповані за автомобілем, відсортовані за пріоритетом. Дія "Прийняти рішення" відкриває {{MODAL:issue_report_actions}}. Пряма дія "Видалити".
- **Налаштування дашборду:** Управління видимістю кожного з 9 віджетів. Зберігається в `localStorage`. Використовує {{MODAL:dash_settings}}.

| Дія | Результат |

### Дії з записом (рядкові)

| Дія | Результат |
| :--- | :--- |
| **Завершити задачу** (чекмарк) | Confirm-запит → POST завершення задачі, оновлення списку |
| **Видалити задачу** (кошик) | Confirm-запит → DELETE задачі, оновлення списку |
| **Прийняти рішення по звіту** (іконка FactCheck) | Відкриває {{MODAL:issue_report_actions}} |
| **Видалити звіт** (кошик) | Confirm-запит → DELETE звіту, оновлення списку |
| **Клік по авто (Статуси)** | Перехід `/assignments?vehicle_id=...` |
| **Клік по авто (Критичне ТО)** | Перехід `/vehicle-maintenance?vehicle_id=...` |
| **Клік по авто (Документи)** | Перехід `/documents?search=[держ.номер]` |
| **Додати задачу** (кнопка `+` у хедері віджета або у порожньому стані) | Відкриває {{MODAL:task_form}} |
| **Переглянути всі задачі** (посилання внизу віджета) | Перехід `/tasks` |
| **Переглянути всі звіти** (посилання внизу віджета) | Перехід `/issue-reports` |
| **Приховати онбордінг-картку** (кнопка `×`) | Ховає конкретну картку онбордінгу (зберігається в `localStorage`) |

## Технічні особливості

> Цей розділ призначений для розробників.

- **Синхронізація:** Компонент `DataUpdateNotice` сигналізує про оновлення даних раз на 10 хвилин (відображає сповіщення, але не перезавантажує автоматично — лише показує підказку). Статус парку (`useParkSetupStatus`) кешується на 5 хвилин без рефетчу при фокусі вікна.
- **Адаптивність:** На мобільних (`useIsMobile`) рендериться `MobileNavigationMenu` замість бічного меню. Хедер показує бургер-кнопку.
- **Права доступу:** Доступна всім авторизованим користувачам без окремого permission. Системний парк (`user.park.is_system === true`) отримує редирект на `/system/dashboard`.
- **Інше:** Налаштування дашборду зберігаються в `localStorage` за ключем `dashboard_settings_v1`. При появі нових віджетів у `DASHBOARD_WIDGETS` — вони автоматично додаються до `enabledWidgets` (merge з DEFAULT_DASHBOARD_CONFIG відбувається при ініціалізації). Онбордінг-картки мають подвійну умову показу: системна (`setupStatus`) + приховування користувачем (`hiddenOnboardingCards` в localStorage).
