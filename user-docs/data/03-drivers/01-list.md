# Список водіїв

{{SCRN:page_drv_list_main}}

> Сторінка дозволяє переглядати, додавати, редагувати та видаляти водіїв парку, керувати їхніми статусами, Telegram-підключенням та призначенням автомобілів.

## Огляд

Сторінка є центральним реєстром водіїв парку. Відображає список водіїв з поточним статусом, призначеним автомобілем, станом інтеграцій (Telegram, Uklon) та коментарями. Дозволяє здійснювати пошук, фільтрацію та сортування. Пов'язана з модулем призначень (`/assignments`) та фінансовим модулем через автоматичне формування записів при призначенні/знятті автомобіля.

## Реєстр даних

| Колонка | Опис | Логіка / Сортування |
| :--- | :--- | :--- |
| **Водій (Телефон)** | ПІБ водія та номер телефону | Сортується за `full_name`. Клік на рядок → відкриває {{MODAL:drv_driver_details}} |
| **Статус** | Поточний робочий статус водія | Сортується за `status`. Значення змінюється через dropdown безпосередньо в таблиці |
| **Telegram** | Стан підключення до Telegram-бота | Не сортується. Клік на значок → відкриває {{MODAL:drv_telegram}} |
| **Uklon** | Наявність зв'язку з акаунтом Uklon | Сіра іконка — не зв'язано. Кольорова — зв'язано. Клік → UklonDriverInfoModal |
| **Автомобіль** | Поточно призначений автомобіль | Відображає держномер та марку/модель. Якщо не призначено — порожньо |
| **Коментарі** | Список коментарів, прив'язаних до водія | Клік на коментар → редагування. Кнопка "+" → додавання нового |
| **Дії** | Контекстне меню | Три крапки → розгортає список дій з записом |

## Функціональні можливості

- **Додавання водія:** Відкриває {{MODAL:drv_driver_form}} у режимі створення.
- **Редагування водія:** Відкриває {{MODAL:drv_driver_form}} у режимі редагування. Доступно з контекстного меню рядка або з {{MODAL:drv_driver_details}}.
- **Перегляд деталей:** Клік на рядок або пункт меню "Показати деталі" відкриває {{MODAL:drv_driver_details}}.
- **Видалення водія:** Пункт контекстного меню "Видалити". Вимагає підтвердження у діалозі.
- **Зміна статусу:** Dropdown безпосередньо в колонці "Статус" — дозволяє змінити статус без відкриття модалки. Виклик `PATCH /drivers/{id}/status`.
- **Призначення автомобіля:** Через {{MODAL:ops_assignment_form}}, що відкривається з контекстного меню (якщо водій без авто) або з {{MODAL:drv_driver_details}}.
- **Telegram:** Відкриває {{MODAL:drv_telegram}} для генерації коду підключення або відключення бота.
- **Імпорт з Excel:** Відкриває {{MODAL:drv_import}} (кнопка "Імпорт" у хедері).
- **Експорт в Excel:** Кнопка "Експорт" у хедері (відображається тільки якщо список не порожній). Завантажує файл через `GET /drivers/export`.
- **Посилання для реєстрації:** Копіює унікальне посилання в буфер обміну — водій може самостійно зареєструватися за цим посиланням.
- **Копіювання в інший парк:** Пункт "Копіювати в інший парк" у контекстному меню. Відкриває {{MODAL:drv_driver_copy}}.
- **Коментарі:** Додавання та редагування коментарів з колонки "Коментарі" через {{MODAL:drv_comments}}.
- **Пошук:** Рядок пошуку у хедері — пошук за ПІБ та телефоном.
- **Фільтрація:** Кнопка фільтрів у хедері відкриває бічну панель. Фільтр: статус водія (динамічно з бекенду).
- **Пагінація:** Відображається при наявності водіїв. За замовчуванням 15 записів на сторінку. Підтримує вибір кількості.

## Технічні особливості

> Цей розділ призначений для розробників.

- **Валідація:** Виконується на стороні сервера. Помилки повертаються у вигляді об'єкту `{field: [messages]}` і відображаються під відповідними полями форми.
- **Синхронізація:** При успішному збереженні/видаленні водія автоматично інвалідується кеш `driversKeys.lists()`. При зміні деталей водія — також `driversKeys.detail(id)`. Кнопка "Копіювати в інший парк" відображається лише якщо в `AuthContext` є декілька доступних парків.
- **Конфлікт призначення:** При спробі призначити автомобіль, що вже зайнятий, сервер повертає `409 Conflict`. У такому випадку відображається додатковий діалог підтвердження з пропозицією перепризначити автомобіль (force assign).
- **Адаптивність:** При ширині < 768px рендериться `DriverListMobile` (карткове відображення). При ширині ≥ 768px — `DriverListDesktop` (таблиця з прилипаючим хедером).
- **Права доступу:** Permission-рядок `drivers.view`. Приховується з навігації через `canAccess('drivers')`. Кнопка "Копіювати в інший парк" відображається тільки якщо `availableParks.length > 1`.
- **Фільтри:** Конфігурація фільтрів завантажується асинхронно через `filterOptionsService.getOptions('drivers')`. До завершення завантаження відображається `Loader`. Стан фільтрів зберігається в URL та `localStorage` (ключ `driversFilters`).
- **Посилання для реєстрації:** Отримується з відповіді `filterOptionsService` у полі `registration_link`. Кнопка disabled якщо посилання відсутнє.
