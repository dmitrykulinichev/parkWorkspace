# Задачі

{{SCRN:page_tasks_main}}

> Сторінка призначена для централізованого управління завданнями парку, відстеження їх виконання та координації роботи команди.

## Огляд

Розділ "Задачі" дозволяє створювати, призначати та контролювати виконання робіт, пов'язаних з автомобілями або загальними справами парку. Задачі інтегровані з модулем "Звіти про проблеми" (Issue Reports), що дозволяє автоматично або вручну створювати завдання на основі звернень водіїв. Система підтримує пріоритетність, відстеження робочого часу та гнучку фільтрацію.

## Реєстр даних

Основний список задач представлений у вигляді таблиці з підтримкою сортування.

| Колонка | Опис | Логіка / Сортування |
| :--- | :--- | :--- |
| **Задача** | Назва задачі, короткий опис та дата створення | Сортування за заголовком. Клік на рядок -> {{MODAL:task_details}} |
| **Статус** | Поточний стан (Нова, В роботі, Завершена тощо) | Використовує випадаючий список для швидкої зміни статусу |
| **Пріоритет** | Важливість задачі | Колірне маркування (Низький, Середній, Високий, Критичний) |
| **Звіти** | Кількість пов'язаних звітів про проблеми | Колір іконки залежить від кількості та критичності звітів |
| **Автомобіль** | Закріплений транспортний засіб | Відображає картку авто. Сортування за ID автомобіля |
| **Призначено** | Виконавець задачі | Картка користувача системи. Сортування за ID виконавця |
| **Створив** | Автор задачі | Картка користувача, який створив запис |

## Функціональні можливості

- **Управління статусами:** Можливість змінювати стан задачі прямо у списку або через деталі.
- **Зв'язок зі звітами:** Відображення активних скарг водіїв, що стали підставою для створення задачі.
- **Швидке завершення:** Спеціальна дія для закриття задачі з фіксацією витраченого часу та фінальним коментарем.
- **Фільтрація та Пошук:** Повнотекстовий пошук за заголовком та описом, а також фільтри за статусом, пріоритетом, автомобілем та виконавцем.
- **Адаптивність:** Мобільна версія пропонує компактний перегляд карток задач зі збереженням ключових дій.

## Технічні особливості

> Цей розділ призначений для розробників.

- **Синхронізація фільтрів:** Фільтри зберігаються в URL та `localStorage` (через `useUrlFilters`), що дозволяє зберігати стан при перезавантаженні.
- **Миттєве оновлення:** Після зміни статусу або видалення список оновлюється автоматично без перезавантаження сторінки завдяки інвалідації кешу React Query.
- **Валідація:** Форма створення задачі має вбудовану обробку серверних помилок валідації з автоматичним скролом до проблемного поля.
- **Пагінація:** Використовується MUI `TablePagination`. Підтримує як стандартний формат `meta`, так і застарілий `pagination` у відповіді API.
