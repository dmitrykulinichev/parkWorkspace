# Графік роботи

{{SCRN:page_drv_schedule_main}}

> Сторінка призначена для планування та візуалізації робочого графіку водіїв, управління призначеннями на автомобілі та контролю фактично відпрацьованих змін.

## Огляд

Графік роботи є центральним інструментом операційного управління парком. Він дозволяє диспетчеру бачити завантаженість автопарку на часовій шкалі (7 днів, 10 днів або Місяць), оперативно призначати водіїв через механізм Drag & Drop та порівнювати планові призначення з фактичними заїздами. Система автоматично перевіряє наявність конфліктів (перетин водіїв або авто) та дозволяє гнучко редагувати розклад.

## Реєстр даних

Основне представлення даних — матриця призначень.

| Колонка / Елемент | Опис | Логіка / Сортування |
| :--- | :--- | :--- |
| **Автомобіль** | Назва та держномер автомобіля (ліва панель) | Сортування за порядком, визначеним у системі |
| **Дата (колонки)** | Дні вибраного часового періоду | Кожна колонка — окремий день |
| **Слот (зміна)** | Клітинка в матриці, що містить призначення | Клік -> {{MODAL:drv_schedule_slot_form}} |
| **Пул водіїв** | Бічна панель (Desktop) зі списком доступних водіїв | Відображає ПІБ та кількість призначених змін за період |

### Стани інтерфейсу

| Стан | Умова відображення | Скріншот |
| :--- | :--- | :--- |

## Функціональні можливості

- **Управління періодами:** Перемикання між тижнями або перехід до поточного дня ("Сьогодні").
- **Налаштування вигляду:** Вибір відображення 7, 10, 14 днів або повного місяця.
- **Drag & Drop призначення:**
    - **З пулу:** Перетягування водія з бічної панелі в клітинку календаря.
    - **Переміщення:** Перетягування існуючого слота в іншу клітинку.
    - **Заміна (Swap):** Перетягування на вже зайнятий слот (у центр) для обміну місцями.
    - **Вставка (Insert):** Перетягування на край клітинки для вставки зміни "До" або "Після" існуючої.
    - **Копіювання:** Утримування `Ctrl` при перетягуванні створює копію слота.
- **Клонування змін:** Копіювання вибраного призначення на вказану кількість днів вперед через контекстне меню або форму.
- **Розділення змін (Split):** Додавання ще одного (порожнього) слота на той самий автомобіль у той самий день.
- **Перегляд фактичних змін:** Візуалізація реальних заїздів водіїв (зелені індикатори) з можливістю перегляду деталей через {{MODAL:drv_actual_shift_details}}.
- **Контроль минулого часу:** Спеціальний перемикач (Lock/Unlock) для захисту від випадкового редагування графіку за минулі дати.
- **Швидкий перегляд водія:** Перегляд короткої картки водія без переходу в основний розділ через {{MODAL:drv_simple_driver_details}}.
- **Експорт в Excel:** Генерація XLSX-файлу з трьома аркушами: повний графік, підсумок по водіях (кількість змін) та статистика завантаженості авто.

## Управління (Actions)

### Глобальні дії (хедер сторінки)

| Дія | Результат |
| :--- | :--- |
| **Стрілки (Вліво/Вправо)** | Навігація по часовій шкалі |
| **Сьогодні** | Повернення до поточного періоду |
| **Вибір періоду (7/10/Міс)** | Зміна масштабу сітки |
| **Експорт** | Завантаження звіту `driver_schedule_YYYY-MM-DD.xlsx` |
| **Замок (Switch)** | Перемикання режиму редагування минулих періодів |

### Дії з записом (слотом)

| Дія | Результат |
| :--- | :--- |
| **Клік на порожню клітинку** | Створення нового призначення через {{MODAL:drv_schedule_slot_form}} |
| **Клік на водія у слоті** | Редагування існуючого призначення |
| **Клік на "Додати зміну" (+)** | Виконання функції Split (додавання порожнього слота) |
| **Клік на індикатор факту** | Перегляд деталей фактичного заїзду через {{MODAL:drv_actual_shift_details}} |
| **Контекстне меню (Клонувати)** | Копіювання на Х днів вперед |
| **Видалення (Кошик)** | Видалення призначення (потребує підтвердження) |

## Технічні особливості

> Цей розділ призначений для розробників.

- **Валідація:** Бекенд перевіряє конфлікти при створенні/переміщенні слотів. Фронтенд візуально підсвічує активні дії.
- **Синхронізація:** Використовується `react-query` для миттєвого оновлення матриці після мутацій.
- **Drag & Drop:** Реалізовано на базі `@dnd-kit`. Дані про перетягування зберігаються у `window.__draggedData`.
- **Експорт:** Використовується бібліотека `xlsx` для формування багатосторінкових звітів на клієнті.
- **Адаптивність:** На мобільних пристроях Пул водіїв приховується, а сітка календаря адаптується під вузький екран.
- **Права доступу:** Доступ регулюється пермішеном `driver-schedule`. Редагування минулого обмежене на рівні API та UI перемикачем `allowPastEditing`.
