# Графік роботи водіїв

{{SCRN:page_drv_schedule_main}}

> Сторінка дозволяє керувати розкладом роботи водіїв, створювати призначення на автомобілі та переглядати фактичні зміни.

## Огляд

Сторінка представляє собою матрицю призначень, де по одній осі розташовані автомобілі, а по іншій — дні періоду. Користувач може створювати, редагувати та видаляти призначення (слоти), а також переглядати інформацію про фактично відпрацьований час водіями.

## Вкладки

### Desktop (Графік)

{{SCRN:page_drv_schedule_main}}

> Відображає повноформатну матрицю графіка.

#### Реєстр даних

| Колонка | Опис | Логіка / Сортування |
| :--- | :--- | :--- |
| **Автомобіль** | Назва автомобіля | Сортування за номером або парком |
| **Дата (колонки)** | Дні вибраного періоду | Клік -> створення слоту |

#### Функціональні можливості

- **Створення слоту:** Відкриває {{MODAL:drv_schedule_slot_form}}.
- **Редагування:** Клік на існуючий слот -> {{MODAL:drv_schedule_slot_form}}.
- **Перегляд деталей:** Клік на фактичну зміну -> {{MODAL:drv_actual_shift_details}}.

#### Управління (Actions)

| Дія | Результат |
| :--- | :--- |
| **Стрілки (період)** | Перемикання тижнів |
| **Drag & Drop** | Переміщення або копіювання слотів |
| **Експорт** | Завантаження графіку в Excel |

#### Технічна інформація

| Файл | Роль |
| :--- | :--- |
| `resources/js/pages/DriverSchedule/ScheduleViews/DriverScheduleDesktop.jsx` | Десктопне представлення |

| Стан | Умова відображення | Скріншот |

## Функціональні можливості

- **Управління періодами:** Перемикання між тижнями (попередній/наступний/поточний).
- **Drag & Drop:** Можливість перетягувати слоти між датами або авто.
- **Блокування редагування минулих періодів:** Перемикач для дозволу редагування минулих дат.
- **Експорт в Excel:** Генерація звіту по поточному графіку.

## Управління (Actions)

### Глобальні дії (хедер сторінки)

| Дія | Результат |
| :--- | :--- |
| **Стрілки навігації** | Перехід по періодах |

| **Кнопка "Експорт"** | Завантаження .xlsx файлу |

## Технічні особливості

- **Валідація:** Перевірка на перетин водіїв та авто в часі.
- **Синхронізація:** Автоматичне оновлення матриці при успішних мутаціях.
- **Права доступу:** Доступ до редагування залежить від ролі користувача та налаштування `allowPastEditing`.
