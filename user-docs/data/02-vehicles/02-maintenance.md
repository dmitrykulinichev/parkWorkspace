# Журнал технічного обслуговування (ТО)

{{PAGE:page_maint_list_main}}

Централізована система обліку всіх сервісних подій автопарку. Дозволяє фіксувати планові регламентні роботи (ТО) та позапланові ремонти, контролювати витрати на обслуговування та зберігати підтверджуючі документи (чеки, акти).

## Функціональні можливості

- **Пакетна реєстрація робіт:** Можливість об'єднати декілька операцій (напр. заміна масла + ремонт ходової) в один заїзд на СТО.
- **Контроль витрат:** Автоматичне підсумовування вартості запчастин та послуг у межах заїзду.
- **Документальний архів:** Завантаження скан-копій або фото документів до всього заїзду або окремих його етапів.
- **Автоматизація пробігу:** Кожен сервісний запис оновлює поточний показник одометра автомобіля.
- **Історія сервісу:** Повний архів обслуговування для кожної одиниці техніки.

### Специфічні стани інтерфейсу
- **Порожній журнал:** Відображається, якщо сервісних подій ще не зафіксовано. {{PAGE:page_maint_list_empty}}
- **Мобільна версія:** Перегляд у вигляді вертикальної стрічки подій. {{PAGE:page_maint_list_mobile}}

## Операційне управління

### Глобальні дії
- **Додати запис ТО:** Відкриває універсальну форму {{MODAL:modal_maint_form_main}}.
- **Пошук та фільтри:** Глобальний пошук по ТЗ та розширена фільтрація ({{MODAL:modal_maint_filters}}).

### Дії з об'єктом
1. **Деталі:** Повний перегляд структури заїзду ({{MODAL:modal_maint_details}}).
2. **Редагувати:** Зміна даних, додавання робіт або файлів.
3. **Видалити:** Видалення заїзду та всіх пов'язаних транзакцій (потребує підтвердження).

## Технічні особливості
- **Синхронізація з фінансами:** Створення запису ТО автоматично генерує видаткову транзакцію в категорії "Ремонт та ТО".
- **Валідація:** При заповненні форми ТО ({{MODAL:modal_maint_form_valid}}) система перевіряє логічну послідовність пробігу.
- **Групування:** Дані в таблиці агрегуються на основі `group_id` для відображення пакетних робіт.
