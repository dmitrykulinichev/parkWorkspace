# Технічне обслуговування

{{SCRN:page_maint_list_main}}

> Централізований журнал для обліку всіх сервісних подій, ремонтів та регламентних робіт автопарку.

## Огляд

Модуль технічного обслуговування (ТО) забезпечує повний контроль над станом транспортних засобів та витратами на їх утримання. Система дозволяє фіксувати як планові заїзди на сервіс, так і неочікувані ремонти, об'єднуючи їх у пакетні записи (групи) для зручності обліку.

Дані цього модуля тісно інтегровані з фінансовим блоком та системою моніторингу пробігу, що забезпечує автоматичне формування витрат та оновлення показників одометра автомобіля при кожному заїзді.

## Функціональні можливості

- **Пакетна реєстрація робіт:** Дозволяє об'єднати декілька сервісних операцій в один заїзд на СТО через {{MODAL:maint_record_form}}.
- **Контроль витрат:** Автоматичне підсумовування вартості всіх робіт та запчастин у межах однієї групи (заїзду).
- **Документальний архів:** Прикріплення скан-копій або фото документів до всього заїзду або окремих робіт.
- **Оновлення пробігу:** Система автоматично оновлює поточний пробіг автомобіля на основі даних, внесених у журнал ТО.
- **Швидкий перегляд деталей:** Перегляд повної структури робіт та вартості без переходу на іншу сторінку через {{MODAL:maint_record_details}}.

## Технічні особливості

> Цей розділ призначений для розробників.

- **Валідація:** При збереженні перевіряється, щоб вказаний пробіг не був меншим за останній відомий пробіг автомобіля.
- **Синхронізація:** Створення або видалення групи записів про ТО автоматично ініціює створення/видалення відповідної видаткової транзакції.
- **Адаптивність:** На мобільних пристроях таблиця трансформується у вертикальний список карток.
- **Групування:** Дані в реєстрі базуються на сутності `MaintenanceGroup`, що агрегує декілька записів `MaintenanceRecord`.
- **Права доступу:** Доступ до сторінки та операцій з даними контролюється пермішеном `vehicles.maintenance`.
