# Виконання нормативів

{{SCRN:tab_rep_reports_performance}}

> Аналітичний звіт для оцінки ефективності автомобілів (KPI) за довільні проміжки часу.

## Огляд

Вкладка "Виконання нормативів" фокусується на порівнянні фактичних результатів роботи автопарку з встановленими плановими нормативами за будь-який обраний період. На відміну від оперативної статистики, цей звіт призначений для ретроспективного аналізу та оцінки стабільності виконання KPI.

## Реєстр даних

| Колонка | Опис | Логіка / Сортування |
| :--- | :--- | :--- |
| **Автомобіль / Дата** | Ідентифікація авто та деталізація по днях | Відображення держномера, ознаки пріоритетності та назви нормативу. |
| **Водіїв / Простій** | Статистика по персоналу та ремонтах | Кількість унікальних водіїв та сумарні дні простою в ремонті. |
| **Вал (Всього)** | Сумарний дохід за період | `Uklon + Ручні`. |
| **Вал (%)** | Відсоток виконання повного плану | Порівняння фактичного доходу з повним планом за період. |
| **Поїздки (Всього)** | Сумарна кількість замовлень | Загальна кількість поїздок за період. |
| **Поїздки (%)** | Відсоток виконання плану поїздок | Відношення факту до плану по кількості замовлень. |

## Функціональні можливості

- **Гнучкий вибір періоду:** Використання `DateRangeFilter` для вибору тижня, місяця або довільного діапазону дат.
- **Деталізація по днях:** Можливість розгорнути статистику кожного автомобіля до рівня конкретних дат та водіїв.
- **KPI Індикація:** Кольорове кодування відсотка виконання (зелений >= 100%, жовтий >= 80%, червоний < 80%).
- **Порівняння з планом:** При наведенні на відсоток виконання з'являється підказка з точними сумами або кількістю поїздок за планом.

## Управління (Actions)

### Глобальні дії (панель фільтрів)

| Дія | Результат |
| :--- | :--- |
| **Вибір діапазону дат** | Оновлення даних звіту за вказаний період. |
| **Деталізація** | Масове розгортання/згортання поденних даних. |
| **Експорт** | Завантаження звіту в Excel. |

## Технічні особливості

> Цей розділ призначений для розробників.

- **Розрахунок плану:** План у цьому звіті є статичним для обраного періоду (повна сума нормативу), на відміну від оперативної статистики, де він пропорційний часу.
- **Мета-дані:** Звіт отримує від бекенду об'єкт `meta`, що містить інформацію про кількість днів у періоді для коректного розрахунку простоїв.
