# Налаштування автопарку

{{PAGE:page_fleet_settings_main}}

Розділ для глобальної конфігурації параметрів роботи автопарку. Основним інструментом тут є управління регламентними роботами, які визначають правила технічного обслуговування для всіх автомобілів парку.

## Регламенти ТО (Schedules)

{{PAGE:page_maint_schedules_list}}

Регламент — це набір правил, за якими система автоматично розраховує термін наступного обслуговування. Наприклад, "Заміна масла кожні 10 000 км".

### Функціональні можливості
- **Двофакторний контроль:** Можливість задавати інтервали одночасно за пробігом (км) та часом (роки). Система спрацює по тому фактору, який настане раніше.
- **Розумні нагадування:** Гнучке налаштування попередження (напр. за 500 км або 14 днів до дедлайну).
- **Сегментація за двигуном:** Регламенти можуть бути загальними або специфічними для типів палива (напр. заміна газових фільтрів лише для авто з ГБО).
- **Пріоритезація:** Визначення критичності робіт, що впливає на сортування у звітах.

### Реєстр регламентів (Таблиця)
| Колонка | Опис |
| :--- | :--- |
| **Назва** | Найменування сервісної операції. |
| **Інтервал (км)** | Періодичність за пробігом. |
| **Інтервал (роки)** | Періодичність за часом. |
| **Тип двигуна** | Для яких авто діє правило (Бензин, Дизель, Газ, Всі). |
| **Пріоритет** | Ступінь важливості (Критичний, Високий, Нормальний). |
| **Статус** | Перемикач активності правила. |

## Керування
- **Додати роботу:** Створення індивідуального правила через {{MODAL:maint_schedule_form}}.
- **Імпорт:** Швидке завантаження стандартних пакетів регламентів через {{MODAL:maint_schedule_import}}.
- **Редагування:** Зміна інтервалів або пріоритетів.
- **Активація/Деактивація:** Тимчасове вимкнення регламенту без його видалення.

## Технічна інформація (для розробників)

| Файл | Роль |
| :--- | :--- |
| `resources/js/pages/FleetSettings/index.jsx` | Головний контейнер з табами налаштувань. |
| `resources/js/pages/MaintenanceSchedules/index.jsx` | Компонент управління регламентами. |
| `resources/js/hooks/useMaintenanceSchedulesPage.js` | Хук управління станом списку та фільтрів регламентів. |
| `resources/js/hooks/useSettings.js` | Глобальні методи CRUD для системних налаштувань. |
| `resources/js/services/vehicleMaintenanceService.js` | Отримання опцій типів двигунів. |

## Технічні особливості
- **Embedded Mode:** Компонент регламентів може працювати як самостійна сторінка або як вбудована частина налаштувань.
- **Синхронізація статусів:** Зміна статусу регламенту в цьому розділі миттєво перераховує стан "Здоров'я" всіх авто у Звітах.
