# Технічне обслуговування (Регламенти)

{{SCRN:tab_rep_reports_maintenance}}

> Звіт для моніторингу статусу регламентних робіт та планування технічного обслуговування автопарку.

## Огляд

Вкладка "Технічне обслуговування" забезпечує візуальний контроль за поточним ресурсом автомобілів (пробіг та час до наступного ТО). Вона дозволяє швидко виявляти автомобілі, що потребують негайного обслуговування, та керувати пріоритетністю заїздів на сервіс.

## Реєстр даних

| Колонка / Елемент | Опис | Логіка / Сортування |
| :--- | :--- | :--- |
| **Картка авто** | Загальна інформація про стан авто | Відображає назву, поточний пробіг та рівень критичності. |
| **Статистична панель** | Зведені метрики по парку | Показує кількість авто з різними рівнями терміновості (Прострочено, Критично тощо). |
| **Параметри ТО** | Індикатори ресурсу для кожного регламенту | Відображає залишок (в км або днях) для кожного налаштованого шаблону ТО. |

## Функціональні можливості

- **Аналіз ТО:** Моніторинг залишку ресурсу по кожному регламенту (км/дні).
- **Кольорова індикація:** Автоматичне розділення станів на "Нормальний", "Увага", "Критичний" та "Прострочено".
- **Звернення:** Відображення кількості активних скарг від водіїв на технічний стан.
- **Пріоритезація:** Система автоматично піднімає вгору списку автомобілі з найбільш критичними показниками зносу запчастин.
- **Пошук та сортування:** Можливість швидкого знаходження автомобіля за номером та сортування за кількістю проблем або пріоритетністю.

## Управління (Actions)

### Глобальні дії (хедер вкладки)

| Дія | Результат |

| **Оновити** | Примусове оновлення даних звіту з сервера. |
| **Скинути фільтри** | Очищення всіх критеріїв пошуку, статусу та сортування. |

### Дії з записом

| Дія | Результат |

## Технічна інформація

> Цей розділ призначений для розробників.

| Файл | Роль |
| :--- | :--- |
| `resources/js/pages/Reports/MaintenanceReportTab.jsx` | Компонент вкладки звіту ТО. |
| `resources/js/hooks/useReports.js` | Хук для отримання звітів та управління фільтрами. |

## Технічні особливості

> Цей розділ призначений для розробників.

- **Кастомні хуки:** Вся логіка фільтрації, сортування та стану звіту інкапсульована в `useReports`.
- **Кольорова схема:** Логіка відображення статусів заснована на метаданих регламентних робіт, що приходять від сервера.
