# Оперативна статистика

{{SCRN:tab_rep_reports_operational}}

> Звіт для моніторингу фінансових та операційних показників у реальному часі з автоматичним прогнозуванням результатів.

## Огляд

Вкладка "Оперативна статистика" надає детальну інформацію про дохід та кількість поїздок кожного автомобіля за поточний день, тиждень або місяць. Система автоматично порівнює фактичні результати з пропорційним планом (на поточний момент часу) та будує прогноз виконання плану до кінця обраного періоду.

## Реєстр даних

| Колонка | Опис | Логіка / Сортування |
| :--- | :--- | :--- |
| **Автомобіль** | Держномер та марка автомобіля | Клік на номер → перехід до картки авто. Відображає теги та індикатор пріоритетності. |
| **Водіїв / Простій** | К-ть унікальних водіїв та днів простою | Сума за період. |
| **Вал (Uklon / Ручні)** | Розподіл доходу за джерелами | Дані з інтеграції та ручних транзакцій. |
| **Вал (Факт)** | Сумарний дохід автомобіля | `Uklon + Ручні`. |
| **Вал (% Вик.)** | Відсоток виконання плану на зараз | Порівняння з пропорційним планом. Колір залежить від % (червоний < 75%, зелений > 90%). |
| **Вал (Прогноз)** | Очікуваний дохід на кінець періоду | Екстраполяція поточної середньодобової виручки. |
| **Поїздки** | Аналогічні показники для кількості поїздок | Порівняння факту, плану та прогнозу за кількістю замовлень. |

## Функціональні можливості

- **Вибір періоду:** Перемикання між "День", "Тиждень", "Місяць". При зміні періоду автоматично перераховується Time Progress.
- **Деталізація по днях:** Розгортання рядка автомобіля для перегляду статистики по кожному дню та водію, що працював у цей день.
- **Індикація ризиків:** Пріоритетні автомобілі з прогнозом поїздок нижче плану підсвічуються червоним кольором та іконкою попередження.
- **Експорт даних:** Завантаження поточного вигляду звіту у форматі Excel.
- **Візуалізація прогресу часу:** Смуга прогресу, що показує, яка частина періоду вже минула (наприклад, 4-й день тижня — 57%).

## Технічні особливості

> Цей розділ призначений для розробників.

- **Прогноз (Forecast):** Розраховується як `(Факт / Минуло_днів) * Всього_днів_в_періоді`.
- **Пропорційний план:** План на поточний момент розраховується як `Повний_план * Time_Progress`.
- **Time Progress:** Використовує `dayjs.isoWeek()` для точного визначення меж тижня за ISO стандартом (понеділок-неділя).
- **Пріоритетні авто:** Логіка `isPriorityTripRisk` спрацьовує, якщо `is_priority && forecast_trips < full_plan_trips - 0.5`.
