# Звіти та Аналітика автопарку

{{PAGE:page_reports_main}}

Центральний аналітичний хаб для моніторингу технічного стану та операційної ефективності парку. Розділ дозволяє менеджеру миттєво ідентифікувати проблемні зони (прострочене ТО, технічні несправності) та прогнозувати витрати.

## Вкладки та розділи

### 1. Технічний стан (Maintenance)
{{PAGE:page_reports_maintenance_tab}}
Спеціалізований звіт для контролю за регламентними роботами кожного автомобіля.
- **Моніторинг розкладів:** Показує залишок кілометрів або днів до наступної заміни масла, фільтрів, ГРМ тощо.
- **Кольорове маркування:** 
    - **Overdue (Червоний):** Роботу вже потрібно було виконати.
    - **Critical (Помаранчевий):** Залишилося менше 10% ресурсу.
    - **Warning (Жовтий):** Наближається термін виконання.
- **Статистика звернень:** Відображає кількість відкритих "Звернень про проблеми" для кожного авто.

### 2. Продуктивність (Performance)
{{PAGE:page_reports_performance_tab}}
Аналіз ефективності використання автомобілів.
- **Пробіги:** Сумарний та середньодобовий пробіг за період.
- **Ефективність:** Порівняння планового та фактичного часу на лінії.

## Функціональні можливості

- **Зведена статистика:** Верхня панель ({{WIDGET:reports_maintenance_stats}}) відображає загальну кількість активних розкладів та частку проблемних авто.
- **Розумне сортування:** Можливість пріоритезації списку за ступенем "критичності" стану (найбільш проблемні авто піднімаються вгору).
- **Пошук:** Миттєвий пошук за держномером або моделлю автомобіля.
- **Фільтрація:** Відсікання справних авто для фокусування лише на тих, що потребують обслуговування.

## Технічна інформація (для розробників)

| Файл | Роль |
| :--- | :--- |
| `resources/js/pages/Reports/index.jsx` | Головний компонент з логікою перемикання табів. |
| `resources/js/hooks/useReports.js` | Основний хук аналітики: завантаження звітів, розрахунок пріоритетів та станів. |
| `resources/js/services/reportsService.js` | Бізнес-логіка: розрахунок інтервалів, визначення "найгіршого" статусу авто, стилізація бейджів. |
| `resources/js/pages/Reports/MaintenanceReportTab.jsx` | Компонент візуалізації звіту по ТО. |
| `resources/js/pages/Reports/PerformanceReportTab.jsx` | Компонент візуалізації продуктивності. |

## Технічні особливості
- **Persistent State:** Параметри фільтрації та обраний таб зберігаються в `localStorage` (через `usePersistentState`), що дозволяє менеджеру повертатися до налаштованого вигляду після перезавантаження сторінки.
- **Пріоритезація:** Алгоритм в `reportsService` автоматично вираховує ступінь критичності ТЗ на основі всіх його активних розкладів ТО та наявних скарг.
