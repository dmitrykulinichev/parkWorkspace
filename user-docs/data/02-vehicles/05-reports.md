# Звіти та Аналітика

{{PAGE:page_reports_main}}

Центральний аналітичний хаб для моніторингу технічного здоров'я та операційної продуктивності автопарку. Розділ дозволяє менеджеру миттєво ідентифікувати проблемні зони (прострочене ТО, технічні несправності) та приймати рішення на основі фактичних даних про пробіги та витрати.

## Функціональні можливості

- **Багатофакторний аналіз ТО:** Моніторинг залишку ресурсу по кожному регламенту (км/дні) з автоматичною пріоритезацією.
- **Кольорова індикація критичності:** Візуальне розділення станів автопарку на "Нормальний", "Увага", "Критичний" та "Прострочено".
- **Операційний контроль:** Аналіз фактичних пробігів та завантаженості автомобілів за обрані періоди.
- **Зведена статистика:** Верхні віджети метрик ({{SEC:reports_maint_stats}}) для миттєвого розуміння загального стану парку.
- **Розумна фільтрація:** Можливість відсікати "здорові" автомобілі, щоб фокусуватися на тих, що потребують термінової уваги розробників.

## Структура Звітів (Таби)

### 1. Технічний стан (Maintenance)
{{PAGE:page_reports_maint_tab}}
Детальний звіт по всіх активних регламентах кожного автомобіля.
- **Список робіт:** Відображає назву регламенту та залишок (напр. "Заміна масла: -500 км").
- **Звернення:** Показує кількість відкритих "Issue Reports" від водіїв.

### 2. Продуктивність (Performance)
{{PAGE:page_reports_perf_tab}}
Аналіз роботи автопарку за часові проміжки.
- **Кілометраж:** Сумарні пробіги за день/тиждень/місяць.
- **Ефективність:** Співвідношення часу в роботі до часу в простої.

## Операційне управління

### Глобальні дії
- **Перемикання табів:** Швидка зміна аналітичного зрізу.
- **Пошук:** Миттєвий фільтр за держномером або назвою авто.
- **Фільтрація ({{MODAL:modal_rep_filters}}):** Налаштування сортування (за пріоритетом, назвою або кількістю проблем) та фільтр за статусом здоров'я.

### Специфічні стани
- **Порожня база:** Відображається, якщо немає даних для розрахунку аналітики. {{PAGE:page_reports_empty}}
- **Мобільний вигляд:** Звіти трансформуються у компактні картки з критичними метриками. {{PAGE:page_reports_mobile}}

## Технічна інформація (для розробників)

| Файл | Роль |
| :--- | :--- |
| `resources/js/pages/Reports/index.jsx` | Головний компонент з логікою табів. |
| `resources/js/hooks/useReports.js` | Основний хук аналітики: завантаження даних, фільтрація та сортування. |
| `resources/js/services/reportsService.js` | Бізнес-логіка: розрахунок інтервалів, стилізація статусів, визначення "Worst Status". |
| `resources/js/pages/Reports/MaintenanceReportTab.jsx` | Візуалізація технічного звіту. |
| `resources/js/hooks/usePersistentState.js` | Збереження налаштувань фільтрів у localStorage. |

## Технічні особливості
- **Автоматична пріоритезація:** Алгоритм в `reportsService` автоматично піднімає вгору списку автомобілі з найбільшим "негативним" пробігом за регламентами.
- **Синхронізація фільтрів:** Налаштування пошуку та сортування зберігаються між сесіями користувача.
- **Адаптивність:** Складні таблиці аналітики перебудовуються у вертикальні стеки на мобільних пристроях.
