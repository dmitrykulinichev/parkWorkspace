# Звіти та Аналітика

{{PAGE:page_reports_main}}

Центральний аналітичний хаб для моніторингу технічного здоров'я та операційної продуктивності автопарку. Розділ розділений на три спеціалізовані вкладки, кожна з яких забезпечує контроль окремого аспекту життєдіяльності парку.

## 1. Технічний стан (Maintenance)
{{PAGE:page_reports_maint_tab}}
Вкладка для оперативного контролю за справністю автомобілів та термінами сервісного обслуговування.

- **Аналіз ТО:** Моніторинг залишку ресурсу по кожному регламенту (км/дні).
- **Кольорова індикація:** Автоматичне розділення станів на "Нормальний", "Увага", "Критичний" та "Прострочено".
- **Звернення:** Відображення кількості активних скарг від водіїв на технічний стан.
- **Пріоритезація:** Система автоматично піднімає вгору списку автомобілі з найбільш критичними показниками зносу запчастин.

## 2. Оперативна статистика (Operational)
{{PAGE:page_reports_oper_tab_main}}
Інструмент для контролю виконання фінансових та операційних планів у реальному часі за обраний період (день, тиждень, місяць).

### Ключові показники:
- **Вал (Дохід):** Розділення на дохід з Uklon та ручні поїздки.
- **Виконання плану (% Вик.):** Відсоткове відношення фактичного доходу до пропорційного плану на поточний момент.
- **Прогнозування:** Розрахунок очікуваного результату до кінця періоду на основі поточної динаміки.
- **Поїздки:** Контроль кількості виконаних замовлень у порівнянні з нормативами.

### Функціональні можливості:
- **Деталізація по днях:** Можливість розгорнути рядок автомобіля для перегляду статистики в розрізі кожного дня та конкретного водія. {{PAGE:page_reports_oper_tab_details}}
- **Система ризиків:** Пріоритетні автомобілі, що не виконують план, підсвічуються спеціальним індикатором тривоги. {{PAGE:page_reports_oper_tab_risk}}
- **Експорт:** Можливість миттєвого завантаження звіту в Excel.

## 3. Продуктивність (Performance)
{{PAGE:page_reports_perf_tab}}
Аналіз загальної ефективності використання активів за тривалі проміжки часу.
- **Пробіги:** Аналіз сумарного кілометражу.
- **Ефективність:** Співвідношення робочого часу до часу простою в ремонті.

## Технічна інформація (для розробників)

| Файл | Роль |
| :--- | :--- |
| `resources/js/pages/Reports/index.jsx` | Головний контейнер з логікою перемикання табів. |
| `resources/js/hooks/useReports.js` | Хук для вкладки Maintenance. |
| `resources/js/queries/reports.js` | Запити `useOperationalReport` та `usePerformanceReport`. |
| `resources/js/services/reportsService.js` | Логіка розрахунку інтервалів та стилів для ТО. |
| `resources/js/pages/Reports/OperationalReportTab.jsx` | Складна таблиця з логікою прогнозування та деталізації. |

## Технічні особливості
- **Динамічний план:** План по доходу розраховується пропорційно часові, що минув (Time Progress), для об'єктивної оцінки виконання.
- **LocalStorage:** Параметри фільтрації (період, деталізація) зберігаються автоматично.
