# Недоступності автомобілів

{{PAGE:page_unavail_list_main}}

Журнал обліку періодів, протягом яких транспортні засоби виведені з експлуатації. Система дозволяє фіксувати причини простоїв (ремонт, ТО, конфіскація) та автоматично керувати операційним статусом автомобіля для запобігання помилковим призначенням у графіку.

## Функціональні можливості

- **Контроль стану активів:** Автоматичне блокування можливості виходу автомобіля на лінію, якщо він знаходиться у статусі "Недоступний".
- **Типізація простоїв:** Розділення за категоріями: Ремонт, Технічне обслуговування (ТО / Огляд) або Тимчасова недоступність (Конфіскований).
- **Розрахунок тривалості:** Автоматичне обчислення часу перебування в простої (дні, години, хвилини).
- **Миттєве розблокування:** Можливість повернути авто в роботу одним кліком через процедуру закриття запису.
- **Хронологічний аудит:** Зберігання повної історії всіх простоїв по кожній одиниці техніки.

## Операційне управління

### Глобальні дії
- **Відправити авто:** Реєстрація нового випадку через {{MODAL:modal_unavail_form}}.
- **Пошук та фільтри:** Швидкий пошук за автомобілем або типом недоступності ({{MODAL:modal_unavail_filters}}).

### Дії з записом
1. **Деталі:** Повний перегляд інформації та нотаток ({{MODAL:modal_unavail_details}}).
2. **Редагувати:** Зміна початкових даних (причини або часу початку).
3. **Закрити (Unlock):** Операція повернення авто в роботу через {{MODAL:modal_unavail_close}}.
4. **Видалити:** Видалення помилкового запису (доступно лише для закритих періодів).

## Технічні особливості
- **Візуалізація:** Активні записи (ті, що тривають зараз) підсвічуються м'яким помаранчевим фоном для швидкої ідентифікації проблемних активів.
- **Валідація:** Система блокує створення дублюючих активних недоступностей для одного ТЗ. При помилках валідації відображається {{MODAL:modal_unavail_valid}}.
- **Синхронізація:** При закритті запису статус автомобіля в системі автоматично змінюється на "Вільний".
