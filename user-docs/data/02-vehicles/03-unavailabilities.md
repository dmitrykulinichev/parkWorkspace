# Недоступності автомобілів

{{PAGE:page_fleet_unavailability_list}}

Журнал обліку періодів, протягом яких автомобілі не можуть використовуватися на лінії. Цей інструмент дозволяє фіксувати причини простоїв, відстежувати їх тривалість та автоматично керувати статусом автомобіля в системі.

## Функціональні можливості

- **Фіксація причин:** Чітке розділення простоїв на типи: Ремонт, Технічне обслуговування (ТО / Огляд), або Конфіскація (напр. штрафмайданчик).
- **Автоматичне управління статусом:** При відправці автомобіля в "недоступність" його системний статус автоматично змінюється на відповідний, що блокує можливість призначення водія або виходу на зміну.
- **Контроль тривалості:** Система автоматично розраховує час перебування в простої (дні, години, хвилини).
- **Повернення в роботу:** Процес "закриття" недоступності повертає автомобіль в активний стан одним кліком.
- **Хронологія:** Повна історія всіх простоїв для аналізу ефективності використання автопарку.

## Операційне управління

### Глобальні дії
- **Відправити авто:** Відкриває форму {{MODAL:fleet_unavailability_form}} для реєстрації нового випадку недоступності.
- **Пошук та фільтри:** Дозволяють знайти записи по конкретному автомобілю або відфільтрувати лише "Активні" (поточні) ремонти.

### Дії з записом
1. **Деталі:** Перегляд розширеної інформації у {{MODAL:fleet_unavailability_details}}.
2. **Редагувати:** Зміна початкових даних (причини або часу початку).
3. **Закрити (Unlock):** Операція повернення авто в роботу через {{MODAL:fleet_unavailability_close}}.
4. **Видалити:** Видалення помилково створеного запису (доступно лише для закритих записів).

## Технічні особливості
- **Візуалізація:** Активні записи (ті, що тривають зараз) підсвічуються м'яким помаранчевим кольором для швидкої візуальної ідентифікації проблемних активів.
- **Валідація:** Система не дозволяє створити нову недоступність для автомобіля, який вже знаходиться в активному стані недоступності.
