# Логи Telegram

{{SCRN:page_intg_tele_logs_main}}

> Сторінка призначена для моніторингу взаємодії системи з Telegram-ботом, перегляду історії отриманих повідомлень та результатів їх обробки.

## Огляд

На цій сторінці відображається детальний журнал запитів, які надходять до системи через Telegram-бота. Дозволяє фільтрувати події за часом, типом дії та статусом виконання, а також переглядати деталі кожного запиту для діагностики проблем.

## Реєстр даних

| Колонка | Опис | Логіка / Сортування |
| :--- | :--- | :--- |
| **Час** | Дата та час отримання запиту. | Хронологічний порядок. |
| **Користувач** | Ім'я, нікнейм (@username) та ID відправника. | Клік відкриває {{MODAL:telegram_log_details}}. |
| **Текст** | Вміст повідомлення або команди. | Відображає текст та (за наявності) відповідь системи. |
| **Тип** | Класифікація типу обробки (тип дії). | Використовується для фільтрації. |
| **Статус** | Результат обробки (Оброблено, Очікує, Помилка). | Візуальна індикація кольором. |

### Стани інтерфейсу

| Стан | Умова відображення | Скріншот |
| :--- | :--- | :--- |
| Порожній | Логи за обраними критеріями відсутні. | {{SCRN:page_intg_tele_logs_empty}} |

## Функціональні можливості

- **Фільтрація логів:** Пошук за текстом/ID, фільтрація за статусом обробки, типом дії та часовим діапазоном.
- **Оновлення даних:** Кнопка для ручного перезавантаження списку логів.
- **Деталізація:** Перегляд повної інформації про запит/відповідь через {{MODAL:telegram_log_details}}.
- **Пагінація:** Управління кількістю записів на сторінці та навігація між сторінками логів.

## Технічні особливості

- **Валідація:** Фільтри (дата) автоматично очищаються при запитах, якщо значення порожні.
- **Статуси:** Помилки критичного рівня позначаються як `is_critical_error`.
- **Дані:** Використовується пагінація з мета-даними (`current_page`, `last_page`, `per_page`, `total`) від API.
