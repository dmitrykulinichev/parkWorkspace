# План реалізації Timeline фактичних призначень

Цей документ описує кроки для додавання горизонтального таймлайну фактичних призначень водіїв під кожним рядком автомобіля в графіку.

## Мета
Візуалізувати реальне використання автомобілів (хто і коли фактично їздив) паралельно з плановим графіком для зручного порівняння.

---

## Крок 1: Бекенд (API)

### 1.1. Новий метод в `DriverScheduleService.php`
Створити метод `getAssignmentTimeline($startDate, $endDate)`, який повертає всі призначення, що перетинаються з вказаним періодом.

**Логіка вибірки:**
```sql
SELECT * FROM vehicle_assignments
WHERE assigned_at <= :end_date
AND (unassigned_at IS NULL OR unassigned_at >= :start_date)
AND driver_id IS NOT NULL
```

### 1.2. Новий метод в `DriverScheduleController.php`
Додати метод `getAssignmentTimeline(Request $request)`, який валідує дати і викликає сервіс.

### 1.3. Маршрут в `routes/api.php`
Додати `GET /driver-schedule/assignment-timeline`.

## Крок 3: Фронтенд (UI Компоненти)

### 3.1. Новий компонент `AssignmentTimelineRow.jsx`
Цей компонент буде відповідати за відображення смужки таймлайну для одного автомобіля.

**Props:**
- `assignments`: Масив призначень для цього авто.
- `startDate`: Початок періоду перегляду (dayjs об'єкт).
- `endDate`: Кінець періоду перегляду (dayjs об'єкт).
- `totalDays`: Кількість днів у періоді.

**Логіка:**
- Розрахунок загальної тривалості періоду в мілісекундах.
- Для кожного призначення розрахунок:
    - `left`: Відступ від початку періоду (%).
    - `width`: Тривалість призначення в межах періоду (%).
    - Обрізка призначень, що виходять за межі періоду.

### 3.2. Новий компонент `AssignmentBar.jsx`
Візуальна смужка призначення.

**Props:**
- `assignment`: Дані призначення.
- `style`: Стилі позиціонування (`left`, `width`).

**Функціонал:**
- Колір фону (колір водія).
- Tooltip з деталями (Ім'я водія, час початку/кінця).

### 3.3. Оновлення `ScheduleCalendar.jsx`
- Імпортувати `useAssignmentTimeline`.
- У циклі рендерингу рядків таблиці (`vehicleIds.map`):
    - Після основного `TableRow` додати новий `TableRow`.
    - Цей рядок має містити одну `TableCell` з `colSpan={weekDays.length + 1}` (або +2, враховуючи колонку авто).
    - Всередині `TableCell` рендерити `AssignmentTimelineRow`.
    - Висота рядка має бути невеликою (наприклад, 10-15px).

## Крок 4: Логіка розрахунку позицій (Деталі)

Для коректного відображення смужок потрібно перетворити час в відсотки.

1. `periodStart` = початок першого дня (00:00).
2. `periodEnd` = кінець останнього дня (23:59:59).
3. `totalDuration` = `periodEnd - periodStart`.

Для кожного призначення:
1. `start` = max(`assignment.assigned_at`, `periodStart`).
2. `end` = min(`assignment.unassigned_at` ?? now(), `periodEnd`).
3. Якщо `end < start`, пропускаємо (призначення поза періодом).
4. `leftPercent` = `(start - periodStart) / totalDuration * 100`.
5. `widthPercent` = `(end - start) / totalDuration * 100`.

## Крок 5: Стилізація
- Таймлайн повинен бути візуально відділений від слотів, але прив'язаний до авто.
- Можливо, варто прибрати бордери між колонками днів у рядку таймлайну, щоб смужка була суцільною.
- Додати легку сітку (вертикальні лінії) на фоні таймлайну, що відповідають дням, для орієнтації.
