## Публічні сторінки

### `register-park.blade.php`
**Призначення:** Публічна сторінка реєстрації нового парку.
**Основні функції:**
- Форма введення назви парку, імені та email власника.
- Автоматичне створення парку, власника та системного бота.
- Відправка листа активації.

### `reset-password.blade.php`
**Призначення:** Публічна сторінка встановлення/скидання пароля.
**Основні функції:**
- Форма введення нового пароля.
- Активація акаунту користувача після встановлення пароля.

## Основні сторінки

### `Dashboard.jsx`
**Призначення:** Головна панель управління, що надає зведений огляд стану автопарку.
**Основні функції:**
- **Пульс парку (New):** Графік у вигляді часової шкали, що відображає ключові події в парку (додавання авто/водіїв, ремонти, звернення) за останні 72 години. Дозволяє візуально оцінити інтенсивність роботи.
- **Onboarding (Welcome Dashboard):** Інтерактивний віджет для нових парків з посиланнями на налаштування (Авто, Водії, Регламенти). Картки автоматично зникають після виконання дій.
- **Vehicle Status List:** Зведена таблиця стану автопарку (В роботі, Вільні, Ремонт).
- **Critical Maintenance:** Віджет критичних нагадувань про ТО з групуванням по типах робіт та детальною інформацією про авто.
- **Document Issues:** Віджет проблем з документами (страховки, ліцензії) з групуванням за статусом (Відсутні, Прострочені).
- **Open Tasks:** Віджет невиконаних задач, згрупованих по автомобілях, з можливістю швидкого завершення або видалення.
- **Open Issue Reports:** Віджет повідомлень про проблеми, згрупованих по автомобілях, з можливістю швидкого прийняття рішень або видалення.
- **Fleet Overview:** Загальна статистика парку (пробіг, витрати).
- Налаштування віджетів (приховування/відображення).
- **Розміщення:** Віджети "Задачі" та "Звіти" розташовані в дві колонки для зручності на десктопі.

### `Profile.jsx`
**Призначення:** Сторінка профілю поточного користувача.
**Основні функції:**
- Редагування особистих даних (ім'я, email).
- Зміна паролю.
- Перегляд призначених ролей та дозволів.
- **Інформація про парк:** Окрема вкладка з даними про поточний парк (тариф, статус).
- **Мульти-парк:** Для користувачів-гастролерів відображається список доступних парків з можливістю перемикання.

### `Statistics.jsx`
**Призначення:** Сторінка аналітики та статистики (демо-версія).
**Основні функції:**
- Графіки загального пробігу, витрат палива.
- Показники ефективності автопарку.
- Топ автомобілів за ефективністю.

## Автомобілі (Vehicles)

### `Vehicles` (Папка)
**Призначення:** Реєстр автомобілів парку.
**Основні функції:**
- Повний список автомобілів з детальною інформацією.
- Фільтрація за статусом, маркою, моделлю тощо.
- Додавання та редагування автомобілів (включаючи нові поля: Рівень комфорту, Кількість місць).
- **Система тегів:** Створення та призначення кольорових тегів для автомобілів (наприклад, "VIP", "Ремонт").
- **Аудит:** Відображення користувача, який додав автомобіль.
- **Масовий імпорт:** Завантаження автомобілів з Excel/CSV файлів з попереднім переглядом та валідацією.
- **Експорт:** Вивантаження списку автомобілів у Excel.
- Оновлення пробігу.
- Швидкі дії: призначення водія, запис на ТО, генерація QR-коду.
- **Налаштування тегів (New):** Модальне вікно для редагування назв та кольорів тегів.

### `IssueReports` (Папка)
**Призначення:** Облік повідомлень про несправності автомобілів (від водіїв або механіків).
**Основні функції:**
- Список проблем з статусами (New, In Progress, Resolved).
- Створення задач на основі звітів про проблеми.
- **Розширені дії:** Прийняття рішень по проблемах (Відхилити, Запросити інфо, Вирішити негайно, Створити задачу).
- **Швидкі дії:** Кнопки "Прийняти рішення" та "Видалити" прямо в таблиці.

### `VehicleMaintenanceRecords` (Папка)
**Призначення:** Журнал технічного обслуговування та ремонтів (Заїзди на сервіс).
**Основні функції:**
- **Групування:** Відображення історії по заїздах (групах робіт).
- **Створення/Редагування заїзду:**
    - Вибір типу роботи: Регламентне ТО або Ремонт.
    - **Smart Repair Input:** Розумний пошук типів ремонтів.
    - Пакетне додавання кількох робіт в одному заїзді.
    - **Фінанси:** Встановлення загальної вартості заїзду (пріоритет) або автоматичний розрахунок суми.
    - **Пробіг:** Встановлення загального пробігу заїзду та (опціонально) індивідуального для робіт.
    - **Коментарі:** Загальний коментар та коментарі до окремих робіт.
    - **Файли:** Прикріплення фото чеків/актів до заїзду та до окремих робіт.
- **Деталі:** Перегляд повної інформації про заїзд, список робіт, файли.
- **Друк:** Генерація PDF-чеку (акту виконаних робіт).
- Фільтрація та пошук.
- **Аудит:** Відображення користувача, який створив запис.

### `MaintenanceSchedules` (Папка)
**Призначення:** Управління регламентами технічного обслуговування.
**Основні функції:**
- Список налаштованих регламентів (інтервали км/рік).
- **Імпорт стандартів:** Завантаження еталонних регламентів з системного каталогу.
    - Dual Listbox інтерфейс (Доступні / Обрані).
    - Відображення статусу синхронізації (Новий, Актуальний, Змінений).
    - Можливість оновлення існуючих регламентів.
    - Синхронізація (видалення) регламентів.
- Створення власних регламентів.
- Фільтрація по типу двигуна.

### `Documents` (Папка)
**Призначення:** Реєстр документів.
**Основні функції:**
- Зберігання та облік документів парку.
- Фільтрація та пошук.
- **Аудит:** Відображення користувача, який створив документ.

### `InsurancePolicies` (Папка) (New)
**Призначення:** Управління страховими полісами автомобілів.
**Основні функції:**
- Реєстр страховок (Автоцивілка, КАСКО, Зелена карта).
- Відстеження термінів дії з автоматичним сповіщенням про закінчення.
- Прикріплення скан-копій полісів.

### `VehicleUnavailabilities` (Папка) (New)
**Призначення:** Облік періодів простою автомобілів.
**Основні функції:**
- Фіксація причин простою (Ремонт, СТО, ДТП, Відсутність водія).
- Календарне відображення доступності всього парку.
- Автоматичне коригування фінансових планів на основі днів простою.

### `Reports.jsx`
**Призначення:** Зведена звітність по автопарку.
**Основні функції:**
- **Оперативна статистика:** Моніторинг виконання планів у реальному часі (День/Тиждень/Місяць) з прогнозами.
    - Таблиця з показниками Вал/Поїздки (Uklon, Ручні, Факт, План, Прогноз).
    - Деталізація по днях: розкриття історії роботи авто за період.
    - Відображення водіїв, що працювали в конкретний день (через слеш, якщо декілька).
    - Відображення днів простою.
    - Відображення тегів автомобілів.
    - **Експорт:** Завантаження звіту в Excel (.xlsx) з повним форматуванням.
- **Технічне обслуговування:** Звіт по регламентних роботах (стан ТО по всьому парку).

## Операційна діяльність (Operations)

### `Drivers` (Папка)
**Призначення:** Реєстр водіїв парку.
**Основні функції:**
- Список водіїв з контактними даними та статусами.
- Додавання та редагування профілів водіїв.
- **Масовий імпорт:** Завантаження списку водіїв з Excel/CSV файлів з валідацією дублікатів (телефон, email, права).
- **Експорт:** Вивантаження списку водіїв у Excel.
- Призначення автомобілів.
- Інтеграція з Telegram (підключення бота).
- Коментарі до водіїв.
- Генерація посилання для реєстрації.

### `VehicleAssignments` (Папка)
**Призначення:** Історія закріплення автомобілів за водіями.
**Основні функції:**
- Журнал призначень (хто, коли, яке авто).
- Створення нових призначень та завершення поточних.
- Фільтрація історії.

### `DriverSchedule` (Папка)
**Призначення:** Планування графіку роботи водіїв.
**Основні функції:**
- Календарний вигляд змін (Desktop/Mobile).
- Drag & Drop функціонал для управління змінами.
- Створення, редагування, копіювання та видалення слотів графіку.
- Експорт графіку в Excel.
- Відображення фактичних змін.

### `Tasks` (Папка)
**Призначення:** Система управління задачами (Task Management).
**Основні функції:**
- Список задач з фільтрацією та пошуком.
- Створення, редагування та видалення задач.
- Зміна статусів задач (To Do, In Progress, Done).
- Детальний перегляд задачі.
- **Аудит:** Відображення користувача, який створив задачу.

## Фінанси (Finance)

### `Transactions` (Папка)
**Призначення:** Облік фінансових операцій.
**Основні функції:**
- Список доходів та витрат.
- Фільтрація за категоріями, датами, авто.
- Створення транзакцій.
- **Аудит:** Відображення користувача, який створив транзакцію.

### `ManualTrips` (Папка)
**Призначення:** Облік ручних поїздок (замовлень поза агрегаторами).
**Основні функції:**
- Список замовлень, доданих вручну або через Telegram-бота.
- Створення, редагування та видалення замовлень.
- Перегляд деталей транзакції, включаючи технічні дані Telegram (Message ID).
- Фільтрація за авто, водієм та датою.
- **Аудит:** Чітке розрізнення записів, створених ботом (автоматично) та менеджером (вручну).

### `DriverExpenses` (Папка)
**Призначення:** Облік витрат водія (гроші, витрачені водієм на потреби парку).
**Основні функції:**
- Список витрат з можливістю фільтрації.
- Створення та редагування записів про витрати.
- **Файли:** Можливість прикріплення фото чеків або актів до запису.
- **Інтеграція:** Автоматичне врахування суми витрат у фінансовому звіті (компенсація водію).
- **Аудит:** Відображення користувача, який створив запис.

### `BonusesPenalties` (Папка)
**Призначення:** Облік премій та штрафів.
**Основні функції:**
- Журнал нарахувань (Премія/Штраф).
- Прив'язка до водія та (опціонально) до автомобіля.
- Фільтрація за типом, датою, водієм.
- Відображення у фінансових звітах.
- **Аудит:** Відображення користувача, який створив запис.

### `DriverEarnings` (Папка)
**Призначення:** Розрахунок заробітної плати водіїв та доходу парку (Базова версія).
**Основні функції:**
- **Звіт:** Тижневий розрахунок зарплати на основі виконаних замовлень (Вал, База, Бонус).
    - **Експорт:** Вивантаження звіту в Excel.
    - **KPI:** Відображення середнього доходу та кількості поїздок за зміну.
    - **Премії та Штрафи:** Окремі колонки з сумами нарахувань.
    - **До видачі:** Фінальна сума з урахуванням заробітку, премій та штрафів.
    - **Парк:** Розрахунок доходу парку ("брудними") після виплати водієві.
- **Схеми:** Створення та редагування тарифних планів (відсотки, плани, бонуси).
- **Налаштування оплати:** Прив'язка індивідуальних схем до водіїв.

### `DriverEarningsNew` (Папка)
**Призначення:** Розширений звіт по заробітку водіїв (Версія 2.0).
**Основні функції:**
- **Зведений звіт:** Детальна таблиця з розділенням показників Uklon та Ручних поїздок.
- **Прозорість:** Відображення бази розрахунку (Profit Total), окремий облік чайових та готівки.
- **Витрати водія:** Окрема колонка з сумою витрат, які компенсуються водію.
- **Фінальний розрахунок:** Автоматичне визначення суми "До виплати" (Зарплата - Готівка + Витрати).
- **Інтерактивність:** Детальні модальні вікна з покроковим розрахунком для кожного водія.
- **Експорт:** Вивантаження деталізованого звіту в Excel.
- **Фільтрація:** Гнучкий вибір періоду (тиждень/місяць/довільний).

### `VehicleEarnings` (Папка)
**Призначення:** Аналіз фінансової ефективності автомобілів.
**Основні функції:**
- **Звіт:** Тижневий розрахунок дохідності авто (Вал, Уклон, Водій, Парк, Витрати, Прибуток).
- **Деталізація:** Перегляд статистики по днях для кожного авто.

## Управління Парком (Park Management)

### `ParkUsers` (Папка)
**Призначення:** Управління співробітниками конкретного парку.
**Основні функції:**
- Додавання менеджерів, диспетчерів, фіндиректорів тощо.
- Призначення ролей в межах парку.
- **Безпека:** Створення користувачів без пароля (через запрошення).
- **Скидання пароля:** Відправка листа або копіювання посилання на активацію.
- **Статус:** Відображення статусу активації акаунту (Активний/Очікує).
- **Аудит:** Відображення інформації про те, хто створив користувача.

### `LandingSettings` (Папка)
**Призначення:** Налаштування публічної сторінки (Лендінгу).
**Основні функції:**
- Редагування текстів, заголовків, контактів.
- Завантаження логотипу та фонових зображень.
- Налаштування SEO та аналітики.

### `Materials` (Папка)
**Призначення:** Друковані матеріали.
**Основні функції:**
- Генерація PDF для друку (QR-коди на авто, листівки для пасажирів, пам'ятки водіям).

### `ActivityLogs` (Папка)
**Призначення:** Аудит дій користувачів.
**Основні функції:**
- Перегляд журналу подій (хто, що, коли змінив).
- Відображення типу дії та ініціатора (включаючи системних ботів).

### `SubscriptionLogs` (Папка)
**Призначення:** Журнал розсилок парку.
**Основні функції:**
- Історія автоматичної відправки звітів (Telegram/Email).
- Моніторинг статусу доставки (Успішно/Помилка).
- Фільтрація за типом звіту, каналом та отримувачем.

### `FinanceSettings` (Папка)
**Призначення:** Налаштування фінансових параметрів.
**Основні функції:**
- Встановлення відсотка комісії агрегатора (Uklon).
- Встановлення базового відсотка водія.

## Системне адміністрування

### `SystemParks` (Папка)
**Призначення:** Управління парками (для Super Admin).
**Основні функції:**
- Створення та редагування парків (телантів).
- **Створення власника:** Можливість вказати ім'я та email власника при створенні парку.
- **Посилання на активацію:** Генерація та відображення посилання для ручної передачі власнику.
- Функція "Impersonate" (вхід від імені парку).

### `SystemUsers` (Папка)
**Призначення:** Управління адміністраторами системи.
**Основні функції:**
- Список системних користувачів.
- **Управління гастролерами:** Надання доступу до кількох парків одному користувачу.
- **Розширене редагування:** Спеціальне модальне вікно для налаштування ролей у різних парках.

### `SystemRoles` (Папка)
**Призначення:** Управління ролями та правами доступу (RBAC).
**Основні функції:**
- Налаштування ролей.

### `SystemSubscriptionPlans` (Папка)
**Призначення:** Управління тарифними планами підписки (SaaS).
**Основні функції:**
- Створення планів (Post-paid модель).
- Встановлення базової абонплати та ціни за активне авто.
- Управління статусами планів (Active/Archived/Inactive).

### `SystemSubscriptionFeatures` (Папка)
**Призначення:** Управління функціональними можливостями для планів.
*   *Примітка: Модуль застарів у новій моделі тарифів, але залишений для сумісності.*

### `SystemSubscriptionLogs` (Папка)
**Призначення:** Системний журнал розсилок.
**Основні функції:**
- Глобальний моніторинг всіх відправлених звітів по всіх парках.
- Інструмент для діагностики проблем з доставкою (Telegram/Email).

### `Debug.jsx`
**Призначення:** Інструменти для розробників.
**Основні функції:**
- Тестування API, часових зон, логування.
- Запуск міграцій БД.
- Тестування інтеграцій.
- **Тестування сповіщень (New):** Форма для відправки тестових сповіщень для поточного або конкретного парку.
