# Документація платформи Park SaaS (Розширена версія)

Цей документ містить детальний опис функціоналу системи, згрупований за ключовими модулями. Інформація базується на фактичній кодовій базі проекту.

## 🚀 Розділ 0: Реєстрація та Тарифи (New)
**Прозорий старт та чесна оплата.**

**1. Публічна реєстрація:**
*   **Самостійний старт:** Можливість зареєструвати новий парк через публічну сторінку `/register-park`.
*   **Автоматичне налаштування:** Система автоматично створює акаунт власника, системного бота та базові налаштування.
*   **Активація:** Власник отримує посилання на встановлення пароля на email (або може отримати його від адміністратора).

**2. Тарифна політика (Post-paid):**
*   **Філософія:** "Спочатку користуєтесь — потім платите".
*   **Структура ціни:**
    *   **База:** Фіксована щомісячна абонплата за доступ до системи (1000 грн).
    *   **Масштаб:** Оплата за кожне **активне авто** (200 грн/міс).
*   **Активне авто:** Автомобіль вважається активним, якщо відпрацював мінімум 5 змін за звітний місяць.
*   **Річна підписка:** Знижка 50% на вартість активного авто при оплаті бази на рік вперед.

## 🔔 Розділ 1: Система Сповіщень (New)
**Миттєве інформування про важливі події.**

**1. Центр сповіщень ("Дзвіночок"):**
*   **Інтерфейс:** У хедері кожної сторінки розташована іконка дзвіночка, яка сигналізує про наявність нових подій.
*   **Лічильник:** Червоний індикатор показує кількість непрочитаних сповіщень.
*   **Список:** При кліку відкривається випадаюче меню з останніми сповіщеннями (прочитаними та новими).

**2. Взаємодія:**
*   **Прочитання:** Клік по будь-якому сповіщенню автоматично позначає його як прочитане.
*   **Масове прочитання:** Кнопка "Прочитати всі" дозволяє очистити лічильник одним кліком.
*   **Дії (Payload):** Сповіщення можуть містити корисне навантаження, наприклад, посилання (`url`). Клік по такому сповіщенню автоматично перенаправить користувача на відповідну сторінку (наприклад, на картку проблемного автомобіля).

**3. Типи сповіщень:**
*   Система підтримує різні типи сповіщень, які візуально відрізняються (Info, Warning, Error, Success).
*   Сповіщення генеруються для всього парку і видимі всім менеджерам, що дозволяє команді спільно реагувати на події.

## 🚕 Розділ 2: Інтеграція з Uklon (Partner API)
**Повна синхронізація з агрегатором.**

Система має глибоку двосторонню інтеграцію з Uklon Fleet API, що дозволяє автоматизувати рутинні процеси обміну даними.

**Реалізований функціонал:**
1.  **Синхронізація Замовлень (Orders Sync):**
    *   Автоматичне завантаження історії поїздок.
    *   Команди синхронізації: `sync-orders` (поточна) та `backfill-orders` (завантаження історії за минулі періоди).
    *   Деталізація замовлення: час подачі, маршрут, вартість, статус виконання.
2.  **Синхронізація Автопарку (Fleet Sync):**
    *   **Link Fleet:** Функціонал для зв'язування автомобілів у локальній базі з автомобілями в кабінеті Uklon.
    *   Імпорт даних про автомобілі з Uklon (`uklon-vehicles`).
3.  **Синхронізація Водіїв (Driver Sync):**
    *   Отримання списку водіїв з Uklon (`uklon-drivers`).
    *   Моніторинг статусу водія в агрегаторі.
4.  **Аналітика (Uklon Reports):**
    *   **Dashboard:** Загальні KPI (Дохід, Поїздки, Пробіг, Середній чек), графік динаміки доходів, діаграма статусів.
    *   **Drivers:** Рейтинг ефективності водіїв (Дохід, Кількість поїздок, Пробіг, Коефіцієнт грн/км).
    *   **Vehicles:** Рейтинг ефективності автомобілів (Дохід, Кількість поїздок, Пробіг, Коефіцієнт грн/км).
    *   Фільтрація за датою та статусами замовлень.
5.  **Технічні особливості:**
    *   Автоматичне оновлення токенів доступу (Refresh Token).
    *   Логування всіх запитів до API для діагностики проблем (`UklonLogs`).
    *   Окремий розділ налаштувань інтеграції (`UklonSettings`).

## 📱 Розділ 3: Екосистема Telegram
**Комунікація з водіями та сповіщення.**

Модуль забезпечує зв'язок між системою управління та персоналом через месенджер.

**Реалізований функціонал:**
1.  **Підключення водіїв:**
    *   Генерація унікальних кодів/посилань для прив'язки Telegram-акаунту водія до його профілю в системі.
    *   Модальне вікно підключення в картці водія.
2.  **Налаштування бота:**
    *   Окремий розділ `TelegramSettings` для конфігурації бота.
    *   Управління токенами та вебхуками.
3.  **Логування:**
    *   Розділ `TelegramLogs` для відстеження історії повідомлень та помилок доставки.
4.  **Сценарії використання (на базі коду):**
    *   Автоматичні сповіщення водіям.
    *   Можливість отримання зворотного зв'язку.
    *   **Додавання замовлень:** Водії можуть самостійно додавати ручні замовлення через бота командою `[номер авто] рука [сума]`.
5.  **Автоматичні звіти (Telegram Reports):**
    *   **Щоденний звіт виконання нормативів:** Відправка зведеної статистики по парку (Дохід, Поїздки, Активні авто) та детальної таблиці по кожному авто за попередній день.
    *   **Щоденний звіт по зарплаті:** Детальний розрахунок заробітку водіїв, каси та суми до видачі.
    *   **Звіт про технічний стан (New):** Комплексний звіт, що включає:
        *   **Критичні ТО:** Прострочені регламентні роботи з деталізацією по авто та типу робіт.
        *   **Проблеми з документами:** Автомобілі з відсутніми, простроченими або такими, що закінчуються, документами (Страховка, Техогляд, Ліцензія).
        *   Відображення тегів автомобілів для швидкої ідентифікації.
    *   **Гібридний звіт:** Зведена статистика за День/Тиждень/Місяць в одній таблиці.

## 🏢 Розділ 4: Управління Парком (Park Management)
**Централізоване керування налаштуваннями та командою.**

**Реалізований функціонал:**
1.  **Команда (Park Users):**
    *   Управління співробітниками парку (менеджери, диспетчери, фіндиректори).
    *   Призначення ролей та прав доступу.
    *   **Безпека:** Створення користувачів без пароля (через запрошення), скидання пароля.
    *   **Аудит:** Відображення інформації про те, хто створив користувача ("Створив").
2.  **Налаштування (Settings):**
    *   **Фінанси:** Встановлення відсотка комісії агрегатора (Uklon), базового відсотка водія та мінімального доходу парку.
    *   **Історія:** Збереження історії змін фінансових налаштувань (ретроспективність).
3.  **Маркетинг та Рекрутинг:**
    *   **Лендінг (Landing Settings):** Конструктор публічної сторінки парку. Налаштування текстів, контактів, логотипу.
    *   **Матеріали (Resources):** Генерація PDF для друку (QR-коди на авто, листівки для пасажирів).
4.  **Журнал подій (Activity Logs):**
    *   Аудит дій користувачів системи (хто, що, коли змінив).
    *   Відображення типу дії та ініціатора (включаючи системних ботів).
5.  **Журнал розсилок (Subscription Logs) (New):**
    *   Історія автоматичної відправки звітів (Telegram/Email).
    *   Моніторинг статусу доставки (Успішно/Помилка).
    *   Фільтрація за типом звіту, каналом та отримувачем.

## 📊 Розділ 5: Головна панель (Dashboard)
**Центр прийняття рішень.**

Агрегатор критично важливої інформації для власника та менеджерів.

**Віджети та функції:**
1.  **Critical Maintenance (Критичне ТО):**
    *   Сучасний дизайн з використанням карток та акордеонів.
    *   Групування нагадувань по типах робіт.
    *   Детальна інформація про авто (назва, номер, пробіг, дата останнього/наступного ТО).
    *   Індикація прострочення (км/дні).
    *   Прямий перехід до картки авто для створення запису ТО.
2.  **Document Issues (Проблеми з документами):**
    *   Моніторинг термінів дії документів (Страховки, Ліцензії, Техогляд).
    *   Групування за типами проблем (Відсутні, Прострочені, Закінчуються).
    *   Прямий перехід до списку авто з фільтрацією по проблемному документу.
3.  **Vehicle Status List:**
    *   Зведена таблиця стану автопарку в реальному часі.
    *   Кількісні показники: Всього авто / В роботі / В ремонті.
4.  **Open Issue Reports (Відкриті звіти):**
    *   Стрічка останніх повідомлень про поломки, що потребують уваги.
    *   **Групування:** Звіти згруповані по автомобілях.
    *   **Індикація:** Кольоровий індикатор пріоритету групи (за найкритичнішим звітом).
    *   **Дії:** Швидкі кнопки "Прийняти рішення" та "Видалити" прямо з віджета.
    *   **Деталі:** Відображення водія, дати та опису проблеми.
5.  **Open Tasks (Відкриті задачі):**
    *   Стрічка невиконаних задач (Open/In Progress).
    *   **Групування:** Задачі згруповані по автомобілях.
    *   **Сортування:** За пріоритетом (критичні зверху).
    *   **Дії:** Швидкі кнопки "Завершити" та "Видалити".
    *   **Зв'язок:** Відображення кількості пов'язаних звітів про проблеми.
6.  **Onboarding (Welcome Dashboard):**
    *   Інтерактивний віджет для нових парків.
    *   Картки з посиланнями на ключові налаштування (Авто, Водії, Регламенти, Інтеграції).
    *   Автоматичне приховування карток після виконання налаштувань (наприклад, картка "Регламенти ТО" зникає після створення першого регламенту).
7.  **Пульс парку (New):**
    *   **Візуалізація активності:** Графік у вигляді часової шкали, що відображає інтенсивність ключових подій в парку за останні 72 години.
    *   **Метрики:** Відстежує додавання авто/водіїв, створення звітів про проблеми, призначення на зміну, ремонти, регламентні роботи, ручні поїздки та оновлення пробігу.
    *   **Динамічна легенда:** Кольори та назви подій налаштовуються на бекенді, що забезпечує гнучкість відображення.

## 📅 Розділ 6: Розумний графік (Driver Schedule)
**Потужний інструмент планування змін.**

Найбільш функціонально насичений модуль для диспетчерів.

**Унікальні можливості (з коду):**
1.  **Матриця планування:**
    *   Відображення: Автомобілі vs Дати.
    *   Підтримка різних масштабів: Тиждень, 10 днів, Місяць.
    *   Навігація між періодами.
2.  **Розширене управління слотами:**
    *   **Drag & Drop:** Перетягування змін мишкою.
    *   **Clone Slot:** Функція копіювання зміни на наступні дні (наприклад, розпланувати водія на тиждень вперед).
    *   **Split Slot:** Розділення слоту (наприклад, додавання другої зміни в той самий день).
    *   **Lock Past Editing:** Функція блокування редагування минулих періодів для захисту історії.
3.  **Типи активності:**
    *   Робоча зміна.
    *   Технічне обслуговування (Maintenance).
    *   Ремонт (Repair).
4.  **Фактичні дані:**
    *   Відображення "Actual Shifts" (фактичних виходів) поруч з плановими.
    *   Модальне вікно деталей фактичної зміни.
5.  **Експорт:**
    *   Генерація `.xlsx` (Excel) файлу з графіком для друку або звітності.

## 🚗 Розділ 7: Автомобілі (Vehicles)
**Повний облік активів.**

**Функціонал:**
1.  **Картка автомобіля (Оновлено):**
    *   **Основні дані:** Держ. номер, VIN, Марка, Модель, Рік.
    *   **Розширені характеристики:** Додано поля "Тип кузова", "Колір", "Рівень комфорту", "Кількість місць", "Типи палива" (мультиселект).
    *   **Creatable Select:** Можливість створення нових марок (Make) та моделей (Model) "на льоту" прямо у формі.
    *   **Дизайн:** Оптимізований компактний вигляд форми (3-4 колонки).
2.  **Масовий імпорт (Новий модуль):**
    *   Підтримка форматів `.xlsx`, `.xls`, `.csv`.
    *   **Інтерфейс:** Модальне вікно з вкладками "Новий імпорт" та "Історія імпортів".
    *   **Попередній перегляд:** Таблиця з результатами валідації файлу перед завантаженням.
    *   **Розумна валідація:**
        *   *Дублікати:* Перевірка по VIN та держ. номеру (підсвічуються помаранчевим).
        *   *Enum значення:* Перевірка кольору, палива, кузова (підсвічуються червоним з підказкою).
        *   *Гнучкий пошук:* Підтримка різних назв колонок (напр. "Make", "Brand", "Марка").
    *   **Історія:** Збереження логів імпорту (Import Batches) з деталізацією по кожному рядку.
    *   **Шаблони:** Динамічна генерація та завантаження шаблонів імпорту.
    *   **Експорт:** Можливість вивантаження повного списку автомобілів у форматі `.xlsx` (Excel).
3.  **Життєвий цикл:**
    *   **Історія пробігу:** Ручне оновлення через модальне вікно `UpdateMileageModal` з історією змін.
    *   **Історія призначень:** Хто і коли керував авто (`VehicleAssignments`).
4.  **QR-ідентифікація:**
    *   Генерація індивідуальних QR-кодів для авто.
    *   Друк стікерів для швидкого доступу до картки авто через смартфон.
5.  **Інтеграція:**
    *   Модальне вікно `VehicleIntegrationModal` для зв'язування з зовнішніми системами.
6.  **Документи:**
    *   Зберігання та облік документів парку.

## 🛠️ Розділ 8: Технічне обслуговування (Maintenance)
**Контроль ремонтів та витрат.**

**Функціонал:**
1.  **Issue Reports (Несправності):**
    *   Система тікетів для фіксації поломок.
    *   Створення задач (Tasks) на основі звітів.
    *   Статуси: New, In Progress, Resolved.
    *   **Дії:** Відхилення, Запит інформації, Негайне вирішення, Створення задачі.
2.  **Maintenance Records (Журнал ТО):**
    *   **Групування (Заїзди на сервіс):** Всі роботи об'єднуються в "Заїзди" (Групи). Один рядок таблиці = Один заїзд.
    *   **Пакетне створення:** Можливість додати необмежену кількість робіт (регламентних або ремонтних) в рамках одного заїзду.
    *   **Фінанси:**
        *   Автоматичний розрахунок загальної вартості заїзду (сума робіт).
        *   Можливість ручного введення загальної вартості (має пріоритет).
        *   Автоматичне створення та оновлення фінансової транзакції для всього заїзду.
    *   **Пробіг:**
        *   Встановлення загального пробігу для заїзду (обов'язково).
        *   Можливість вказати індивідуальний пробіг для окремої роботи (опціонально).
        *   Автоматичне оновлення пробігу автомобіля.
    *   **Файли:**
        *   Прикріплення загальних файлів (чеків, актів) до заїзду.
        *   Прикріплення файлів до окремих робіт.
        *   Відображення всіх файлів у списку та деталях.
    *   **Коментарі:** Загальний коментар до заїзду + індивідуальні коментарі до робіт.
    *   **Друк:** Генерація та друк Акту виконаних робіт (Чеку) прямо з картки заїзду.
    *   **Розумний ввід (Smart Repair Input):** Автокомпліт для типів ремонтів.
    *   **Аудит:** Відображення користувача, який створив запис.

## 💰 Розділ 10: Фінанси (Finance)
**Комплексний облік грошових потоків, автоматичний розрахунок зарплати та аналіз прибутковості.**

Модуль дозволяє автоматизувати фінансові відносини з водіями та бачити реальну економіку кожного автомобіля.

**1. Транзакції (Transactions):**
*   **Журнал операцій:** Єдиний реєстр всіх доходів та витрат парку.
*   **Категоризація:** Розподіл транзакцій за типами (Паливо, Ремонт, Зарплата, Штрафи тощо).
*   **Прив'язка:** Можливість прив'язати транзакцію до конкретного водія або автомобіля для точного обліку.
*   **Аудит:** Відображення користувача, який створив транзакцію.

**2. Ручні поїздки (Manual Trips):**
*   **Облік "кешових" замовлень:** Реєстрація поїздок, виконаних поза агрегаторами (від бордюру, особисті клієнти).
*   **Telegram-бот:** Водії можуть самостійно додавати замовлення через бота командою `[номер авто] рука [сума]`.
*   **Редагування:** При редагуванні повідомлення в Telegram, запис автоматично оновлюється в системі.
*   **Інтеграція:** Дані автоматично враховуються у звітах "Заробіток водіїв", "Заробіток автомобілів" та "Оперативна статистика".
*   **Аудит:** Чітке розрізнення записів, створених ботом (автоматично) та менеджером (вручну).

**3. Витрати водія (Driver Expenses):**
*   **Облік витрат:** Реєстрація коштів, які водій витратив зі своєї кишені на потреби парку (наприклад, заміна лампочки, антифриз).
*   **Файли:** Можливість прикріплення фото чеків або актів до запису.
*   **Компенсація:** Сума витрат автоматично додається до суми "До виплати" у фінансовому звіті.

**4. Заробіток водіїв (Driver Earnings):**
*   **Гнучкі тарифні плани (Financial Schemes):**
    *   Створення необмеженої кількості схем розрахунку (наприклад, "Новачок 50/50", "Профі 70/30").
    *   **Прогресивна шкала:** Можливість встановити тижневий план (наприклад, 23 800 грн). При виконанні плану відсоток водія автоматично підвищується для суми понад план.
    *   **Індивідуальний підхід:** Прив'язка персональної схеми до кожного водія через таб "Налаштування оплати".
*   **Автоматичний розрахунок (Payroll):**
    *   Система автоматично завантажує виконані замовлення з Uklon та додає ручні поїздки.
    *   **Алгоритм:** Вал (Gross) -> Віднімання комісії Uklon -> Розрахунок частки водія (База + Бонус) -> Розрахунок доходу парку.
    *   **Врахування Премій та Штрафів:** Автоматичне додавання бонусів та віднімання штрафів у фінальній сумі "До видачі".
    *   **Звітність:** Детальна таблиця з розбивкою по кожному водію за вибраний період (тиждень/місяць).
    *   **KPI:** Відображення середнього доходу та кількості поїздок за зміну.
    *   **Експорт:** Можливість вивантаження звіту в Excel (.xlsx) з повною деталізацією.
    *   **Деталізація:** Розкриття статистики по днях для кожного водія.

**5. Заробіток водіїв 2.0 (Driver Earnings New):**
*   **Оновлений модуль розрахунку:** Покращена версія звіту з глибшою інтеграцією Uklon.
*   **Деталізація джерел:** Чітке розділення показників по Uklon (Кількість, Вал, Profit Total) та Ручних поїздках.
*   **Прозорість розрахунків:**
    *   **Profit Total:** Використання "чистого" доходу від Uklon як бази для нарахування відсотка водію.
    *   **Чайові:** Окремий облік чайових, які додаються до зарплати водія в повному обсязі (100%).
    *   **Готівка:** Облік готівки, яку водій вже отримав на руки, для коректного розрахунку фінальної виплати.
    *   **Витрати водія:** Врахування компенсації витрат водія.
*   **Фінальний розрахунок (Payout):** Автоматичне визначення суми "До виплати" (Зарплата - Готівка + Витрати).
*   **Інтерактивність:**
    *   **Детальна модалка:** При кліку на рядок відкривається покроковий розрахунок для конкретного водія ("розжовування" формули).
    *   **Інфо-довідка:** Вбудована довідка з поясненням принципів нарахування.
*   **Debug Mode:** Можливість перегляду сирих JSON даних, отриманих з бекенду, для діагностики розбіжностей (кнопка в панелі фільтрів).
*   **Експорт:** Вивантаження деталізованого звіту в Excel.

**6. Заробіток автомобілів (Vehicle Earnings):**
*   **Економіка авто (Unit Economics):** Аналіз прибутковості кожної одиниці техніки.
*   **Формула чистого прибутку:**
    `Прибуток = (Вал - Комісія Uklon - Зарплата водія) - Витрати на авто`
    *   *Зарплата водія:* Для оцінки ефективності авто використовується усереднений показник (наприклад, 65% від чистого валу).
    *   *Витрати:* Автоматично підтягуються всі транзакції типу "Витрата" (ТО, ремонти, запчастини), прив'язані до цього авто.
*   **Деталізація:**
    *   Можливість переглянути статистику по кожному дню: скільки авто заробило і скільки витратило.
    *   KPI картки: Пробіг, Кількість поїздок, Середній чек, Ефективність (Грн/км).

**7. Премії та Штрафи (Bonuses & Penalties):**
*   **Журнал нарахувань:** Окремий реєстр для фіксації фінансових стягнень та заохочень.
*   **Гнучкість:** Можливість прив'язки до водія та (опціонально) до автомобіля.
*   **Інтеграція:** Автоматичне відображення у звіті "Заробіток водіїв" окремими колонками.
*   **Типи:** Чітке розділення на "Премія" (зелений) та "Штраф" (червоний).
*   **Аудит:** Відображення користувача, який створив запис.

**8. Налаштування фінансів (Finance Settings):**
*   **Комісія агрегатора:** Централізоване налаштування відсотка комісії (наприклад, 18%), який Uklon стягує з парку. Це значення автоматично використовується у всіх розрахунках для отримання "Чистого валу" (Net Revenue).

## 📈 Розділ 11: Звітність та Аналітика (Reports)
**Контроль ефективності та виконання планів.**

**1. Оперативна статистика (Operational Stats):**

*   **Пропорційний план:** Автоматичний розрахунок плану на поточний момент часу (наприклад, якщо пройшло 3 дні тижня, план складає 3/7 від тижневої норми).
*   **Прогнозування:** Розрахунок очікуваного результату на кінець періоду на основі поточної динаміки (Вал та Поїздки).
*   **Деталізація по днях:** Розкриття історії роботи авто за період з відображенням конкретних водіїв та їхніх показників.
*   **Облік простою:** Чітке відображення днів, коли авто не працювало (червоний бейдж).
*   **Експорт:** Можливість завантаження звіту в Excel (.xlsx) з повним форматуванням та структурою.
*   **KPI:** Вал (Дохід), Кількість поїздок, Кількість унікальних водіїв.
*   **Чистий дохід:** Всі фінансові показники відображаються з вирахуванням комісії Uklon.

**2. Виконання нормативів (Performance Report):**
*   **Історичний аналіз:** Звіт про виконання планів за будь-який обраний період.
*   **Точний розрахунок:** Врахування кількості днів у кожному місяці для точного розрахунку планових показників.
*   **Деталізація:** Аналогічна до оперативної статистики (по днях та водіях).
*   **Експорт:** Вивантаження історичних даних в Excel.
*   **Візуалізація:** Кольорове кодування відсотків виконання (Зелений/Жовтий/Червоний).

**3. Технічне обслуговування (Maintenance Report):**
*   Зведений звіт про стан регламентних робіт по всьому парку.
