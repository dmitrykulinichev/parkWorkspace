# Стандарт найменування скріншотів (Screenshot Naming Convention)

Цей документ визначає правила назв для всіх графічних матеріалів проєкту Garage24. Дотримання цього стандарту є обов'язковим для розробників, тестувальників та ШІ-асистентів.

## Формула назви
`[тип]_[модуль]_[об'єкт]_[дія/стан].png`

---

## 1. Типи (Prefix)
Визначає, який елемент інтерфейсу зображено:
- `page_` — ціла сторінка (наприклад, реєстр або детальний перегляд).
- `modal_` — модальне вікно або діалог поверх сторінки.
- `widget_` — окремий функціональний блок (наприклад, на дашборді).
- `mobile_` — інтерфейс, знятий у мобільному розширенні.
- `print_` — вигляд документа, підготовленого до друку або PDF.
- `step_` — конкретний крок у багатоетапних процесах (візарди).

## 2. Модулі (Scope)
Визначає приналежність до бізнес-логіки:
- `dashboard` — головна панель.
- `fleet` — автопарк, страховки, техпаспорти.
- `drivers` — водії, HR, профілі.
- `fin` — фінанси, транзакції, заробіток.
- `sch` — графік (schedule), зміни.
- `maint` — технічне обслуговування (maintenance).
- `uklon` — інтеграція з агрегатором.
- `tg` — Telegram бот та налаштування.
- `admin` — налаштування парку, користувачі.
- `sys` — системне адміністрування (Super Admin).

## 3. Об'єкти та Дії
Описують суть контенту:
- `list` — таблиця, реєстр.
- `form` / `create` / `edit` — форми вводу.
- `details` / `view` — детальна інформація про об'єкт.
- `empty` — вигляд сторінки без даних.
- `success` / `error` — стани завершення операції.

---

## Приклади використання

### Автопарк:
- `page_fleet_vehicle_list.png` — головний список авто.
- `modal_fleet_vehicle_create.png` — вікно додавання нового авто.
- `modal_fleet_mileage_update.png` — вікно оновлення пробігу.
- `page_fleet_policy_list.png` — список страховок.

### Фінанси:
- `page_fin_transaction_list.png` — журнал транзакцій.
- `modal_fin_payout_create.png` — створення виплати водію.
- `widget_dashboard_fin_summary.png` — фінансовий віджет на головній.

### Графік:
- `page_sch_matrix.png` — основна сітка графіку.
- `mobile_sch_driver_view.png` — графік очима водія на смартфоні.

---

## Рекомендації до створення:
1. **Якість:** Формат PNG. Бажано без лишніх закладок браузера (тільки робоча область).
2. **Дані:** Використовуйте реалістичні демо-дані. Приховуйте реальні персональні дані (Прізвища, Телефони) якщо це не демо-стенд.
3. **Мова:** Інтерфейс має бути українською (якщо не тестується інша локалізація).
